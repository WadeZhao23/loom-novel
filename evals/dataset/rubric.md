# Judge 维度操作化 Rubric(v1)

判读单位:一章正文(`chapter.md`)+ 该例 context 四键(`setting` / `characters` / `prev_hook` / `chapter_goal`)。Judge 逐维独立判「本章是否命中该硬伤」,各维不合并、不互相抵消。

Judge 只准判本文件声明的这 8 个维度;不许对整章下一个笼统的好坏结论,也不许把结果折算成任何分数(ADR-0002:只做逐维硬伤诊断,不做文学优劣的综合评价)。判读态度继承内置质检 CRITIC 原文:**宁缺毋滥,没把握的不算硬伤**——两可情形一律判「未命中」,把假阳压到最低。

语义基准是 `loom/gates.py` 里 `CRITIC_质检`(硬伤 ①–⑦)与 `CRITIC_去AI味`(AI 腔六类 + 指纹豁免护栏)两段提示词原文。本 rubric 只把原文操作化,不发明原文没有的判据。

每个维度按「定义 → 该抓(正例) → 不该抓(反例) → 边界例 → 严重度 → 证据要求」六节展开。正例/反例/边界例均为具体情节示例,不是定义复述。

---

## 人物OOC

### 定义
CRITIC 原文①:角色违背既定**性格 / 立场 / 已知信息 / 人物硬约束的底线身段**。即人物做出与其人物卡设定的性格、阵营立场、他此刻实际掌握的信息、或不可逾越的行为底线相冲突的言行。

### 该抓(正例)
人物卡写「沈砚寡言、谋定后动」,本章他一进矿道就当众高声嚷嚷,还主动把「逆息体质忌讳外泄」的秘密抖给陌生矿工听——既违寡言性格,又破体质保密的底线身段,可判 OOC。

### 不该抓(反例)
同样是沈砚开口示警,但前文已铺垫他察觉同伴命悬一线、不得不破例出声,正文给足了动机。有合理动机、有铺垫的「破例 / 成长」不算 OOC。

### 边界例
性格突变 vs 有铺垫的成长:一向懦弱的配角本章忽然强硬顶撞上位者。若本章或前情有触发点(亲人被辱、退无可退),视为有铺垫的转变,不抓;若毫无铺垫、突变又突回,判 OOC。两可时依「宁缺毋滥」不抓。

### 严重度
高=违背人物硬约束底线(泄露忌讳秘密、背叛核心阵营)或性格 180 度反转且无铺垫;中=与既定性格明显不符但未触底线(寡言者忽然滔滔不绝);低=语气 / 小习惯与人设轻微出戏。

### 证据要求
chapter 型。`evidence` 引违背处的原文短句(那句出戏的台词或动作),`note` 写清它违背了人物卡的哪一条(性格 / 立场 / 已知信息 / 底线)。

## 设定漂移

### 定义
CRITIC 原文②:**违反世界观 / 金手指 / 已埋伏笔**。即本章描写与 `setting` 世界观规则、主角金手指(能力 / 系统)的既定机制、或前文已埋下的伏笔相矛盾。

### 该抓(正例)
`setting` 定「灵气复苏初期,凡人炼气三年起步」,本章一名路人甲昨日还是凡人、今日已能御剑腾空——违反世界观的修炼节奏硬规则,判设定漂移。

### 不该抓(反例)
本章第一次交代「北境有一处灵脉能就地加速炼气」,这是对既有世界观的扩写补白,与已定规则不冲突、也不推翻金手指边界。新设定扩写不算漂移。

### 边界例
新设定扩写 vs 与已有设定冲突:本章引入一件从未提过的法宝。若其能力落在世界观允许区间、不架空已定规则,算合法扩写,不抓;若它的效果直接否定前文定死的某条规则(如「此界无飞行法宝」),则判漂移。

### 严重度
高=违反世界观核心硬规则或金手指底层机制(修炼体系、力量上限被推翻);中=与前文已埋的伏笔 / 已定支线矛盾;低=设定细节与前文轻微冲突(某物产地、某势力人数前后不一)。

### 证据要求
chapter 型。`evidence` 引漂移处原文,`note` 写清它违反了哪条既定设定(引 `setting` 或前文对应规则)。

## 断钩子

### 定义
CRITIC 原文③:**没接住上一章章末的钩子**。`prev_hook` 抛出的悬念 / 事件,整章正文完全没有回应、承接或推进,读者上一章被吊起的期待落空。这是 absence 型缺陷。

### 该抓(正例)
`prev_hook` 是「矿道尽头传来提灯脚步声」,本章从头到尾写沈砚回屋喝茶、盘账、睡下,通篇不提矿道、不提脚步声,也无任何人物对那声响的反应——钩子被整章无视,判断钩子。

### 不该抓(反例)
本章开篇即写沈砚循声摸向矿道尽头、与提灯人打了照面——哪怕结果是虚惊一场,只要正文正面承接了 `prev_hook`,就不算断钩子。

### 边界例
延迟接钩(本章埋、下章接):本章没直接照面提灯人,但明确写了沈砚对脚步声起疑、决定「今夜先探清退路、明日再查」。有正面回应与推进意图的延迟处理不判;只有连「意识到钩子存在」都没有才判。

### 严重度
高=`prev_hook` 是主线强钩(生死 / 追兵 / 揭底),整章零回应;中=支线钩子被无视;低=次要悬念未接但主钩已接。

### 证据要求
absence 型。缺陷是「缺失」,引不出原文,`evidence` 栏留空;`note` 里写清上一章钩子是什么(转述 `prev_hook`),以及本章为何判为未接(通篇无任何承接 / 回应 / 推进)。

## 无爽点

### 定义
CRITIC 原文④:**整章没有任何爽点 / 收获**。全章读下来,视点人物既无情绪释放(打脸、逆袭、扬眉、悬念揭晓),也无实质进展(实力、信息、资源、关系的正向增量),读者读完一无所得。这是 absence 型缺陷。

### 该抓(正例)
整章就是主角被反派当众羞辱、处处受制、最后灰溜溜离场,既没还手也没留下任何反转伏笔或信息增量,通篇是纯粹的憋屈铺陈——整章无任何收获,判无爽点。

### 不该抓(反例)
本章主角同样全程隐忍未发作,但结尾他默默记下了对方的破绽、顺手拿到一枚关键令牌。有明确的信息或资源增量,哪怕情绪上不爽,也不判无爽点。

### 边界例
铺垫章的「暗爽」算不算收获:一个纯过渡章没有打脸高潮,但读者随主角摸清了敌方布局、拼上一块悬念拼图。这类信息增量式的「暗爽」算收获,不抓;只有连信息 / 伏笔增量都没有的空转章才判。判无爽点从严,有一处实质增量即不抓。

### 严重度
高=整章纯负向(全程受挫)且无任何伏笔 / 信息补偿;中=平淡空转、进展极微弱;低=有收获但爽感偏薄。

### 证据要求
absence 型。爽点是「缺失」,引不出原文,`evidence` 栏留空;`note` 写清本章整体走向,以及为何判定全章无任何情绪释放或实质增量。

## 信息边界

### 定义
CRITIC 原文⑤,**双向**:a) 角色表现得知道他不在场 / 未发生的事;b) 角色展示越境界 / 越阅历的认知,正文却没有一句来源交代(金手指 / 情报 / 前世记忆)——缺归因同样算硬伤。

### 该抓(正例)
- a 款:沈砚整章待在矿道,却在对话里准确说出此刻城主府密室内正在发生的密谈内容,正文没有任何他获知该信息的渠道——知道不在场之事,命中。
- b 款:一个初入炼气的少年张口就道破对手所用上古秘术的克制法门,正文全程没交代他从哪得知(无师承、无情报、无前世记忆)——越阅历认知且无归因,命中。

### 不该抓(反例)
同样是少年点破秘术,但正文写明「他前世曾是此道宗师,残存记忆一闪而过」。有明确的来源归因句(金手指 / 情报 / 前世记忆),b 款不抓。

### 边界例
有归因句就不抓(b 款):主角说出超出其见识的判断,正文哪怕一句带过「系统提示」「探子来报」「师父曾言」,即视为已归因,不判;通篇找不到任何来源交代才判 b 款。a 款(知道物理上不可能知道之事)则即便给了「他猜的」也要看是否合理,不合理仍判。

### 严重度
高=a 款知晓关键剧情信息(反派底牌 / 主角秘密)且直接改变本章走向;中=b 款越阅历认知无归因;低=轻微越界(说出略超身份的常识)。

### 证据要求
chapter 型。`evidence` 引越界认知的原文台词或内心活动,`note` 写清是 a 款(引 `prev_hook`/`setting` 证明该事他不在场)还是 b 款(说明正文通篇无归因句)。

## 物品状态连续性

### 定义
CRITIC 原文⑥:**使用了状态账本或前文回顾里已标「消耗 / 失去」的物品,或人物状态与账本不符**。即本章让一件已用掉、丢失、损毁的物品重新出场,或人物的伤势 / 储备等状态与前情记录相矛盾。

### 该抓(正例)
前情 / 状态账本明确「那枚旧矿牌已在上一章交给守卫、不再持有」,本章沈砚却又从怀里掏出同一枚旧矿牌递人——已标失去的物品复现,命中。

### 不该抓(反例)
本章沈砚掏出一枚矿牌,但正文交代这是他新从矿场领的另一枚。同名不同物、来源清楚,账本上那枚仍记为失去,并不矛盾,不抓。

### 边界例
同名不同物(两把同名刀):本章出现一把与前文已损毁之刀同名的刀。若正文点明是新铸 / 另一把(不同来源或特征),不判;若正文把它当作那把已断之刀的同一件继续使用,则判物品状态矛盾。

### 严重度
高=违反状态账本明确记录的核心物品 / 状态(已消耗的保命丹又吃一次);中=与前文回顾里的物品状态矛盾;低=次要物品或非关键状态的细节不符。

### 证据要求
chapter 型。`evidence` 引本章复现该物品 / 状态的原文,`note` 写清账本或前文把它记为何种状态(消耗 / 失去 / 受伤),与本章如何冲突。

## 时间连续性

### 定义
CRITIC 原文⑦:**时间词与前情时刻粒度不符**(如昨夜发生的事说成昨日)。即本章的时间指代与前文已确立的时间线,在粒度或先后上对不上。

### 该抓(正例)
前情明确矿道遇袭发生在「昨夜三更」,本章沈砚回忆时却说成「上个月那场袭击」——时间粒度与前情严重不符,判时间连续性命中。

### 不该抓(反例)
前情写事件发生在昨夜,本章说「昨夜的事」或「不到一天前」。与既定时刻一致或属合理换算,不抓。

### 边界例
模糊时间词的宽容度:本章用「不久前」「前些日子」指代一件前情发生在三天前的事。模糊词只要不与既定时刻明确冲突(三天前称「不久前」尚在合理区间),从宽不抓;只有模糊词也已越出合理范围(三年前称「不久前」)才判。

### 严重度
高=时间错位改变因果 / 顺序(把后发生的说成先发生);中=时间粒度明显不符(昨夜说成上月);低=模糊时间词略含糊但不改事实。

### 证据要求
chapter 型。`evidence` 引本章那处时间词原文,`note` 写清前情把该事件锚定在何时(引 `prev_hook` 或 `setting`),两者粒度 / 先后如何冲突。

## AI腔

### 定义
去 AI 味 gate 原文:挑出本章终稿里**具体命中**的 AI 腔词句——套话头尾、空洞万能词、过度连接词、直接点名情绪、黑名单词、句式过整齐。只挑具体命中的词句,不作整体文风的印象评断。

### 该抓(正例)
段落以「在这个世界上,每个人都……」的空洞万能句起头,情绪靠「他感到无比的震惊与愤怒」直接点名,句与句用「不仅……而且……因此……」焊死、长短一律——同时命中套话头尾 + 直接点名情绪 + 过度连接词。

### 不该抓(反例)
作者惯用的短促口头禅、略显生硬的重复句式,若与该作者既有文风一致,属个人腔调而非模板腔——不算 AI 腔(见护栏)。

### 边界例
指纹豁免护栏(去 AI 味原文):某处不规整或口头禅,若是【写作指纹】带来的作者个人特征(像他本人),**不算 AI 腔,别挑**。判定两可时先假定是作者指纹而非 AI 腔,依护栏放过——宁可漏挑,也不把作者签名句式误判成 AI 腔。

### 严重度
高=通篇模板腔(套话头尾 + 情绪点名 + 句式整齐叠加,严重出戏);中=局部多处空洞万能词 / 过度连接词;低=零星一两个黑名单词或轻微句式规整。

### 证据要求
chapter 型。`evidence` 引命中的 AI 腔词句原文,`note` 写清命中哪一类(套话头尾 / 万能词 / 连接词 / 情绪点名 / 黑名单词 / 句式过整齐),并确认已排除写作指纹豁免。
