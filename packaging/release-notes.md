把一队分工 Agent 织成一条**写小说的流水线**,做成桌面客户端。读你的"外置大脑"一键跑出一章正文;你手改,它**越写越像你**。你的书和写作指纹都存在本地,Loom 自身不收集、不上传;只有用 AI 写作时,才把必要上下文发给你选的模型(DeepSeek 云端 / 本机 claude·codex / 任意 OpenAI 兼容供应商)。自带 DeepSeek key,按字数计费。

## 0.4.2 有什么

**「除虫」更火眼金睛了。**

Loom 写完一章会自动「除虫」——挑出前后对不上的地方。这版它多认了两类毛病:一是**时间倒流**(前面说「三天后」,这章开头又变回「当天」);二是**本该闭关/被关着的人突然出现**、正文却没交代他怎么出来的。以前这两类容易漏,现在当场标出来给你看。

**顺带:认人更准了。**同一个角色换个简称、去掉姓、或用「姓+称呼」叫,以前除虫可能认不出、跟着漏抓;现在对得上,也不会因为名字像就误伤别人。

老书零迁移。

## 0.4.1 有什么

**提设定不再沉默崩溃。**

以前你把一个不存在的槽位写成落点发给「伙伴」,它不会提醒你——默默收下,等你点确认时才弹出「未知槽位」,你也不知道哪错了。现在它直接告诉你「这个落点不存在,去看地基里复制有效的」,当场就能纠正。

**顺带:**「看地基」的描述里也加了提示——它的返回值就是唯一有效落点列表,不要自己编路径。

## 0.4.0 有什么

**帮你搭书的「伙伴」会聊天了。**

以前建一本新书,得对着一堆空模板一格一格填;现在直接跟它聊——你说想写什么,它给你几个现成方向挑一个,一路把这本书该定的（写什么、主角是谁、世界观、前几章怎么走）帮你一起定下来。

它做事的时候,你**看得见它在想、在干**,不用干等一片空白;觉得跑偏了、啰嗦了,随时能**停**。

## 0.3.9 有什么

**答题起书——新书不再面对一堆空模板。**

打开一本新书,左边多了个「伙伴」面板:它一段一段地问你——这本书发哪个平台、写什么题材、主角是谁、世界观怎么设、前几章怎么走。每道题给你几个现成选项,你挑一个、改一改、或者自己写;你定下的答案**直接落进对应的设定文件**,和你手写的一模一样、随时能改。不想答的段落点「跳过」,想回头改点那一段的名字。答完(或全跳过),面板自己收起来。

它只帮你**问**、绝不替你**编**:选项是给你拍板的提议,你点了才算数,Loom 从不自己发明设定。问什么只看你文件里现在写了什么——所以你在别处改了设定,它下次自然接着现状问。接模型后才会出题;不接也不影响原来的用法。

**顺带修了个 Windows 上的建书 bug**:书名里带 `" : * ? < > |` 这类字符时,以前在 Windows 上建书会直接失败(这些字符不能做文件夹名)。现在书名会自动清成合法的文件夹名,书名本身原样保留、分毫不改。

老书零迁移。

## 0.3.8 有什么

品牌楷体随包，三端长一个样。

logo、弹窗标题、章题这些「品牌时刻」以前用楷体，靠的是你系统里的字体兜底——Windows 的楷体没粗细、非中文系统甚至没楷体，标题就塌了。这一版把**霞鹜文楷屏幕阅读版**的常用字子集（GB2312 全 6763 字，约 1.7MB）直接**捆进包里**，Mac / Windows / 任何系统看到的品牌楷体都一样。正文字体不变（太大、不捆），子集外的生僻字自动回退到系统字体，不影响排版。

纯观感打磨，无功能改动、无破坏性改动；老书零迁移。

## 0.3.7 有什么

「设了 2000 字,写出来还是忽长忽短」——这一版给字数控制拧上几颗确定性的螺丝。

**字数更听话**
- 编辑/润色现在**知道原稿实际多少字**(以前靠模型自己数,数不准):明显超出发布篇幅(±20%)才带着「超 X%」去压回目标量级、不动情节;还在合格范围的稿不瞎压。
- 改了目标字数后,如果还在用**按旧字数拆的细纲**,会提示你「重新生成细纲」——这是改了字数却不听话最常见的原因。
- 细纲没给各场标字数、或总和跟目标差太多,会提醒你篇幅可能失控。
- 超长提醒更早(超目标 1.25 倍就提,不再等 1.5 倍)、更显眼(专门弹一下,不只默默记留痕)。

**说句实话**:AI 没法边写边精确数字数,单章还会有 ±25-30% 的波动——这是模型的天花板。这一版把「数字数」从模型手里拿回代码手里,尽量逼近你设的目标,但做不到分毫不差。想要某章篇幅精确,手改最稳。

Mac / Windows 都适用,无破坏性改动;老书零迁移。

## 0.3.6 有什么

已经用别的工具攒了一堆设定、大纲、人物、章纲?现在**选个文件夹就能接进来**,不用对着空表格重填。

**导入资料夹 · 接成一本新书**
- 欢迎页点「导入资料夹」,选中你放 md 的文件夹——Loom 按文件名归位:大纲/章纲/总纲/细纲/纲要 进卡章纲、人物进人物、设定进世界观;猜不出来的列给你,你点一下指认放哪。
- **原样接进来,不改你一个字**(连全角缩进、空行都照留),也不调 AI 重塑——你的设定还是你的。GBK 旧稿也吃得下、不会坏书。书名默认取文件夹名,可改。
- 导入完给一份小结:各类收了几份;哪些地方要顺手调,一次说清。

**诚实说清两点**
- 境界/专名的逐字保护,靠世界观文件名带「力量体系/地理势力」这类词自动识别;名字太个性化的,导入小结会提示你把它单列成一节(在世界观里选中用「AI 改写」即可)。
- 导入的是**设定**,不是文风;还没正文时写作先用中性文风,写几章、手改后点 learn 会越来越像你。

你那些设定/大纲工具强在前期,Loom 专治**写作**那一段——现在一个文件夹就接上了。

Mac / Windows 都适用,无破坏性改动;老书零迁移。

## 0.3.5 有什么

写长篇最怕的不是写不出,是「第 2 章吞掉的药胚第 4 章又复活了」。这一版给 Loom 装上**除虫闭环**。

**每章写完自动「除虫」**
- 对照前几章、状态账本、设定,查五类跨章矛盾:物品被消耗后复用、人设跌份、金手指规则数值前后不一、时间词粒度错、衔接冲突。报告分严重度、带两处原文证据和**一句可直接用的修改示例**;走便宜模型,可在设置关闭。
- 老书福利:编辑器点「除虫」可对任何旧章节补查——从第 1 章逐章跑一遍,账本自动建起来。
- 报告里点「按建议改」:自动定位到涉事段落,修改示例直接填进行内改写面板,候选你确认才落盘。

**状态账本(脉络新页签)**
- 每章自动记四类状态:物品得失消耗 / 人物境界伤势 / 金手指数值 / 章末故事时间,每条带原文证据。你可改可删,AI 只追加、绝不动你改过的节。
- 写新章时大纲师/写手会收到「当前状态」摘录——已消耗物品与规则数值**有账可对、被逐字盯住**,除虫在终稿后再兜一层。

**生成侧也更稳**
- 人物卡的底线/身段/命格逐字直送写手(金丹天骄不再对炼气期下跪);系统流规则(「系统」「因果」类标题)纳入硬设定逐字保护。
- 质检新增三查:物品连续性、时间连续性、越阶认知缺来源交代会被质检点名;回炉明令「不扩写」。

**诚实局限(先说清楚)**
- 除虫是放大镜不是保票:报告不自动改稿,「按建议改」要你确认才落盘;别名/简称和没入账的状态可能漏;老书从第 1 章逐章点「除虫」补账;重写过的章,删掉账本里对应「## 第N章」节再扫一次。

Mac / Windows 都适用,无破坏性改动;老书零迁移。

## 0.3.4 有什么

写作台从「整章生成器」再往「愿意天天泡在里面的写作工具」走一步——正文和设定都能就地让 AI 改。

**正文随处可改(就地,不再弹大窗)**
- 鼠标移到正文某一行,行边浮出「改」——点开就在那一行下面展开小面板,说一句怎么改(留空=按你的嗓音改顺),候选出来「用这个 / 再来一个」你定;指令框里按 Enter 直接跑。
- 选中一段(跨行也行)照旧浮出朱批条,面板同样就地展开;中央大弹窗退役。
- 铁律不变:AI 改动同步进 .原稿 快照,learn 永远只学你的手改;切文件、换书、手动改文时面板自动收起,候选绝不会落错地方。

**世界观 / 人物也能让 AI 改写、续写**
- 打开世界观或人物文件:选中一段点「改写选中」,或点「AI 续写」顺着现有设定往下铺一节;候选你确认才落盘,不确认不动你一个字。
- 设定改写走设定师口径(与书名、其余设定自洽,不发明冲突),绝不碰你的写作指纹。
- AI 铺完底稿会讲清三种维护方式:随时手改 / 让 AI 改写续写 / learn 后自动追加。

Mac / Windows 都适用,无破坏性改动。

## 0.3.3 有什么

这一版专治「新建一本书,第一章却和书名、想法对不上」——从建书那一刻就把设定铺进去,让第一章一开始就长在你的世界里。老书零迁移、无破坏性改动,升级即生效。

**建书就能定调,AI 一键铺底稿**
- 新建书时可以直接写一句话设定(如「重生成崇祯,开局砍魏忠贤,拯救大明」)、选目标平台;填了设定、模型也接好,进书房就自动按书名 + 设定起草**世界观 / 人物 / 卡章纲**三件套底稿,你改成自己的再开写。
- 起步清单从四步变五步,「AI 铺设定底稿」进了主流程;外置大脑还空着就点「织第一章」会先拦一下、提醒你先铺底稿——直接硬写,AI 只能按题材套路瞎编。

**第一章不再漂移(引擎修复)**
- **模型现在知道这本书叫什么**:书名会写进每一道工序的提示词。以前它对「这本书是什么」一无所知,于是《重生之我是崇祯》能写出个不相干的通用金手指开局。
- **出厂模板不再冒充你的设定**:那些「(占位示例:……)」的填写说明、还空着的表单行,以前会被当成真设定喂给模型、照着执行;现在一律不进提示词——你没填就是没填,AI 按书名 + 题材自由发挥,不照一份错纲跑。
- 历史 / 都市这类题材起草时不再硬套「力量体系 / 金手指」,换成「时代格局 / 主角优势」。

Mac / Windows 都适用,无破坏性改动;老书的外置大脑与写作指纹一字不动。
