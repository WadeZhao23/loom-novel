# Loom

把一队分工 Agent 织成一条写小说流水线的命令行工具。本文件是项目的**词表**——只定义概念,不写实现。

## Language

**外置大脑**:
某一本小说**专属、会随故事生长**的状态文件(世界观、人物卡、卡章纲、写作指纹,以及可选的违禁词、立项卡、文风参考)。
_Avoid_: 记忆库、设定集(指同一概念时统一叫"外置大脑")

**立项卡(定位)** 〔已实现,见 [ADR 0011](docs/adr/0011-project-card-positioning.md)〕:
某本书**起书时人手填、之后基本不动**的定位卡——记 `平台 / 分区 / 题材 / 对标意图 / 为什么选它`,属外置大脑里**人手维护、只读不回写**的那类(同 `违禁词.md`)。各格可空(任一格有内容即算立项完成);**立项/世界观/主角/前几章纲是开写第一章的硬门槛**(见 [ADR 0014](docs/adr/0014-startup-completeness-gate.md)),不是可选。平台、题材由建书代落,其余人手填,loom 不另改。两个用途:① **平台**给 `违禁词.md` 定基线松紧(起点/番茄严、晋江另一套),是全库**唯一**对平台敏感的联动;② **整卡喂设定师**当定位背景(管 what/where,不碰 voice)。
_Avoid_: 塞进 `loom.toml` 当字段(流水线没人读=死数据;定位是给设定师和人看的散文,不是配置);建 `平台→分区→题材` 级联映射树 / 起书时联网实时拉分区(分区几年不动、又无公开 API 只能脆爬虫,还砸 `loom init` 的零联网离线不变量——见 [ADR 0011](docs/adr/0011-project-card-positioning.md));把对标素材原文塞进卡(素材是你另贴给 `拆书` 的本地文本,卡只记「选了谁、为什么」);当它是会随故事长的设定(它不长,没有 `[AI补充]`)

**skills**:
**跨书复用、基本不变**的写作手艺/方法论,能原样拷进任何小说项目。
_Avoid_: 技巧库、方法库(指 `skills/` 目录时统一叫 "skills")

**agents**:
流水线上**干活的一个工序**——由一段系统提示词 + 一份 `reads:` 清单(声明它要拉进哪些外置大脑/skills 文件)构成。
_Avoid_: 用题材名词命名 agent(见下方"Flagged ambiguities")

**领航员** 〔已实现,见 [ADR 0015](docs/adr/0015-navigator-agent.md);继承 [ADR 0013](docs/adr/0013-journey-orchestration-no-langgraph.md) 的手搓/无第二状态真相红线〕:
第六个角色的**人设与红线**——书房伙伴对话 agent 的角色名。架在五后端共用单轮 `complete()` 之上的手搓对话循环(每轮=一次 HTTP 往返,`POST /api/partner/say`,ndjson 流):读书文件现状(谓词/槽位现算)+ 对话尾部拼 prompt,回复解析成说话段 + 文本协议工具块(读文件/看地基/提设定;**一条消息可连发多个「用:提设定」给多方向候选,≤3**,解析器容忍括号壳——FB-B/2026-07-18)。**读类工具执行回喂续跑(≤6次/轮)、提了候选卡本轮即终结交作者拍板**(提完不再 re-complete——真机实测重开会让模型二次质疑、重提重复卡),无工具也终结。红线:只出候选、**绝不替作者做决定**;候选必须走**候选卡**(结构性可见;占位模板不算旧值、拍板按钮统一「落库」;同槽多候选选一个后其余变灰、想换走对话)、作者点了才算数,没点的一个字不进书——这是唯一挡住 AI 文字混进书的结构性保证。加载走项目 `agents/领航员.md` 优先、包内模板回退(老书免升级,人设作者可改、改了立即生效)。
**命名公约**:**领航员**=角色名与盘上命名——`agents/领航员.md`、`.审稿留痕/领航员留痕.md` 一律不改名(改名会让老书自定义人设静默回退包内模板、留痕断档);**(书房)伙伴**=对作者的 UI 称呼(「伙伴近况」、`.伙伴对话/`,与既有「伙伴面板」同词根);**partner**=API/代码层命名(`partner.py`/`partner_context.py`/`partner_tools.py`/`partner_store.py`、`POST /api/partner/*`)。三层各管一层,不互相顶替。
_Avoid_: 让它替作者做决定/未拍板就落盘;把对话历史当状态读回(唯一例外:待确认 proposal 行,见「.伙伴对话」词条);让对话内容回流写作指纹(学习信号仍只是改稿 diff,ADR 0002);跨层混用命名(见上命名公约)

**创作旅程(伙伴面板)**:
起书地基的**阶段谓词层**(`journey.py` STAGES 表:立项→世界观→人物→卡章纲→voice);完成判据全部从外置大脑文件现状派生(门禁/各段完成度现算,不逐章硬查、不存问答历史)。**两个形态渲染同一条对话流**:起书四段未解锁时,书房伙伴占居中大卡(直奔还空着的地基格发问);解锁后搬右下 48px 悬浮球(常驻、随时可召;副标题现算「现在定「X」」引导——分段进度条 chip 已撤,2026-07-18 bug3),两个形态背后是同一条 `.伙伴对话/` 对话流,不是两套状态。织章期间对话照聊,只有拍板落盘与开工序要等本次织章跑完(见「领航员」词条候选卡红线)。**IP**:头像=水墨卡通「提灯小童」(`webui/agents/navigator.jpg`,生成留档见 `docs/design/proposals/navigator-avatar.md`)。**对话体验一期(2026-07-18,活动视图+P1.5+v2,spec 在 `docs/superpowers/specs/2026-07-17-partner-activity-view-design.md`)**:工具执行做成可展开的动作记录(「看了地基 ▸」)、DeepSeek 思考过程实时灰字流(「已思考 ▸」可回看,**transient 不落盘**、getattr 读 reasoning_content、内省守卫只挂声明后端)、回复中可「停」(前端 AbortController + 后端轮边界取消 + 归档轮锁门禁)、落盘后自动引下一格;**老版问答卡切换已下线,对话式是唯一形态**(`partnerUiMode()` 恒 chat)。
_Avoid_: 引入 langgraph/langchain 做这层编排(ADR 0013 留档否决);给旅程建第二状态真相(sqlite/checkpoint);把对话历史当真相读回(完成度/门禁永远从文件推导,唯一例外见「.伙伴对话」词条)

**`.伙伴对话`** 〔已实现,见 [ADR 0015](docs/adr/0015-navigator-agent.md)〕:
书根下(与 `.审稿留痕` 同级,**不进外置大脑**,五道工序永不读它)的对话存储:一条对话一个 `当前.jsonl`(`user`/`assistant`/`tool`/`result`/`proposal`/`confirm`/`meta`/`summary` 事件,单事件单行 append,坏行跳过);「新开一段」把它归档成 `归档-<时间戳>.jsonl`——一本书一条连续对话,伙伴是一个人不是 N 个聊天窗。`loom backup` 自动包含;**整个目录删掉,书完好无损**——未拍板的候选随之作废,伙伴失忆,仅此而已(上下文≠状态)。
_Avoid_: 把它当状态读回派生完成度/门禁/游标(唯一例外:**待确认 proposal 行**,同 ADR 0013 薄游标「待答卡缓存」待遇——过期归档失效/重复确认幂等/落点快照失配拒绝三条判据兜底);进外置大脑(会被 agent 读到污染设定);当持久执行上下文(两轮之间进程零挂起,每轮从这份 jsonl 尾部 + 文件现状重建,不依赖任何内存里活着的执行栈)

**写作指纹**:
对"你"的文风的**规则化描述**(含若干条逐字保留的你的原句作 **anchor 例句**);一个会随你对生成稿的手改而演进的**活文件**,属于外置大脑。**目的只有一个:让生成稿带上你的个人嗓音(像你)**。每本小说**独立一份**。
_Avoid_: 把它理解成"你写的句子本身"(它是对句子的描述);写作风格、style(统一叫"写作指纹");给它绑"过检测/检测分数"(已撤回)

**播种 (seed)**:
`loom seed` 提炼出初版写作指纹的动作(**独立命令、联网**);来源可以是你贴的**2–3 段真实样本**、**你欣赏的作者的原文范文**(L1「借力起点」——开局像他,此后靠你的改稿脱化;详 [ADR 0012](docs/adr/0012-reference-others-influence-voice.md)),也可以是**你另一本书的写作指纹**(继承起点,此后各自独立演进)。`loom init` 只离线铺一份**中性默认指纹**,不播种。
_Avoid_: 空白模板填写(已废弃:那不是"你");把种子当**持续学习信号**(种子只定**起点**、一次性;写作指纹的学习信号永远只是你的改稿=学习增量,故即便从别人样本起手也会朝你收敛——这就是刻意不走 L3 的原因)

**导入铺底(资料夹→新书)** 〔已实现于 0.3.6〕:
把创作者手里一堆**非-Loom 的散装 md**(设定/人物/大纲/章纲,来自别的写作 skill/工具)一次性接成**一本新 Loom 书**的动作:选文件夹 → 按**文件名启发**路由(大纲/章纲类→`卡章纲.md` 拼接、人物/角色类→`人物/`、其余设定→`世界观/`)、**猜不中的列出来让作者指认**落哪个桶 → 生成 `loom.toml` + **原样落盘** → 出一份**导入小结**(几份落哪 + 降级提示)。**纯机械搬运:不调 LLM、不重塑他的内容**(守「loom 不发明设定、不改写人写的主体」)。书名默认取文件夹名、可改——把"创作新书还得重填表格"变成"确认预填"。是继 `draft`(空手起草)/`seed`(voice 起点)/`拆书`(what 参考) 之后**第四条**往外置大脑灌料的通道,方向独特:「你已有一大堆料,我只**归位**、不无中生有、不碰 voice」。导入现也吞**正文章节**(`.txt`/`.md`,按章序重排为 `第N章.md`,内容原样);导入章无 AI 快照、不能 `learn` 属正常(见「整书诊断」词条,可从这些正文提炼设定回填外置大脑)。
_Avoid_: 让 AI 把异构 md **重塑**成 Loom 结构(硬设定小节/一行章纲)——刻意**不做**:有失真风险、违机械纯净、且用户实报最怕 AI 改坏他的料;导入即让**硬设定直送/自动记忆**挂上(他的标题/格式不合就是不挂,靠**导入小结明示降级** + 作者事后用「世界观 AI 改写」手动归位,不在导入时硬塞);自动填 `立项卡`/`违禁词`(人手维护红线,只搬他现成的、没有就留模板);合并进**已有**书(v1 只做资料夹→**新**书,合并的去重/覆盖语义另说);和 `draft` 混为一谈(draft 从一句话无中生有铺三件套;导入是从他已有的料归位,不发明)

**整书诊断**:
进书后伙伴面板的独立动作「从正文提炼设定」——`diagnose.py` 读采样正文(前 3 + 最近 2 章)、cheap LLM 把**正文里已有**的主角/世界观/章纲提炼成候选(**逐条带第N章出处**,有出处=提炼、无出处=发明),**确认前不落盘**;作者一屏勾选/改写 → 复用人写优先落盘器写外置大脑(跳过 _digest;撞人写成品进访谈补充,绝不覆盖);主角指认落 `主角·名字.md`。落盘后签名失配、领航员只问真缺、门禁自动开。**立项段永不进诊断**(正文无平台/对标意图,守 ADR 0011 不回写)。
_Avoid_: importer 里调 LLM(零 LLM 红线,诊断是导入后的独立动作);直接 atomic_write 外置大脑(必先候选+确认);诊断碰立项卡;LLM 选正文切点(归拆章工具)

**学习增量 (human delta)**:
你的**改稿 vs AI 原稿的 diff**;是写作指纹**唯一**合法的学习信号。
_Avoid_: 从 AI 终稿回写(已禁止:会让指纹收敛到 AI 自己的腔)

**工作区(本章工作区)**:
一次 `loom write` 跑动期间,按工序顺序**累积**各 agent 产物(设定锚点→细纲→初稿→改稿→终稿)的中间状态;每个 agent 读**到目前为止的整个工作区**。
_Avoid_: 纯链式(只读上一步)——已否决,会丢设定锚点导致设定漂移

**硬设定直送(逐字旁路)**:
设定师把世界观压成 ≤350 字「设定锚点」是**语义选择**(这章用哪几条),但**境界等级、金手指代价、地名、人物专名**这类**必须逐字不变**的硬事实经"设定师锚点→大纲师细纲"两道复述会写漂(F~SSS 凭空多出"一阶0级"、"一中"写成"二中")。故在锚点之外,把世界观这些小节连同人物专名册**确定性切片、原文**直送大纲师/写手(`agents._hardfacts_for`),绕开复述。纯字符串切片,不调 LLM/不打分/不建实体库;缺文件返空、绝不阻断出稿(详 [ADR 0010](docs/adr/0010-hardfacts-verbatim-passthrough.md))。
_Avoid_: 把**冰山真相**(终极反转)逐字喂写手(会提前剧透——反转段 deny 压过硬设定关键词);整本世界观塞给写手(稀释写作指纹、且夹带反转);拿它替代设定师锚点的"选择"职责(两者互补:锚点选这章用哪几条 what、硬设定保专名/等级不漂);把它折进续跑账本签名(源文件已在设定师 reads 里,改它必从第0棒重跑)

**细纲(可看可改)**:
大纲师把一章拆成**分镜骨头**,场次数**以本章字数预算为准**(短章 2-3 场、长章 4-6 场,不用多场把写手篇幅撑爆;曾硬编码「3-6 场」,字数五螺丝去掉)。落盘到 `正文/.细纲/第N章.md`、**可看可改**:一旦这文件存在,大纲师就**直接读它**(WYSIWYG)——你改了它,重写本章就按你的来;想要新方案,清空它或点「重新生成细纲」(跑 设定师→大纲师 覆盖它,此刻吃到最新的卡章纲/世界观/人物卡)。删章时随整章产物进回收站(见 [ADR 0008](docs/adr/0008-scene-outline-editable-wysiwyg.md))。
_Avoid_: 和**卡章纲**混为一谈(卡章纲=一章一句话的人写脊柱+`[AI回顾]`;细纲=这一章的详细分镜);自动累积成"全书分镜册"(无跨章价值,跨章记忆靠卡章纲 `[AI回顾]`)

**AI 原稿快照**:
生成第 N 章时存进 `正文/.原稿/第N章.md` 的那份 AI 终稿;**唯一用途**是给 `loom learn` 跟你手改后的 `正文/第N章.md` 做 diff(即算出学习增量)。
_Avoid_: 拿它当"上一章"喂连续性——连续性读**手改后的** `正文/第N章.md`,不读快照

**写后摘要(AI回顾)**:
`loom learn` 接受一章后,从**手改终稿**蒸馏出的 ≤150 字摘要 + 伏笔三态(埋设/推进/回收),write-once 回填到卡章纲该章行下的 `[AI回顾]` 缩进子块,补跨章记忆。只管 what,绝不回流写作指纹。
_Avoid_: 把它喂给写手当文风;覆盖人手写的卡章纲规划行(它只追加子块)

**外置大脑随章生长([AI补充])**:
`loom learn` 接受一章后,从**手改定稿**里蒸馏出**这章新冒出来、且现有文件里还没记下**的世界观设定/人物信息,以物理隔离的 `[AI补充·第N章]` 块 write-once **追加**到 `世界观.md` / `人物卡.md` 末尾的「AI 补充」区。和 [AI回顾] 同源同红线:只追加、绝不覆盖人手写的主体、只管 what、不喂写作指纹;删章时连同 [AI回顾] 一起清掉本章的 [AI补充](见 [ADR 0007](docs/adr/0007-external-brain-grows-with-story.md))。
_Avoid_: 让 AI 就地改写人手写的世界观/人物卡正文(只在文末追加);把它喂给写手当文风;自动回写**大纲**(向前规划仍归人,只向后的回顾/补充自动长)

**题材速查**:
`skills/题材/<题材>.md`,一份"这个题材要什么套路(核心爽感/读者期待/必备桥段/雷区/金手指)"的跨书手艺;新建书时按选的题材**只拷一份**给设定师。管 what。
_Avoid_: 把题材套路 reads 进写手/润色师/写作指纹(会把"像你"污染成"像那个题材")

**拆书**:
`loom deconstruct <本地文本>`(**CLI、单份**)把你自己找的一本参考作品拆成**可迁移条件框架**(什么情境造出爽点/张力),强制把专名(境界/人名/地名)剥进**黑名单**,**绝不碰 voice**。产物落隔离的 `外置大脑/.拆书/{名}-拆解.md`——**没有任何 agent 读它、`run_pipeline` 跳过**;你**手工**把想要的框架挑进 `世界观.md`、每条带一句「差异化改造」。管 what,是"参考别人"的 **what 侧**通道。
_Avoid_: 指望它自动收集/发现/排序对标(只吃你贴的单份,收集归人);拿产物当 voice 源喂写手(那是 `文风参考` 的活,拆书刻意**去** voice、正好相反);让 agent 直接读 `.拆书/`(隔离死信,不手工挑进世界观 + 差异化改造就是抄)

**文风参考** 〔已实现,见 [ADR 0012](docs/adr/0012-reference-others-influence-voice.md)〕:
一份**可选**的外置大脑文件,持有你贴的 2–3 段**欣赏作者的原文范文**;每写一章都进 `写手` 的 `reads:` 当**风格样例**垫着,是"参考别人"的 **voice 侧**通道(L2)。与 `拆书` 恰成一对:拆书抽 what·**剥** voice·隔离没人读;文风参考留 prose·**喂** voice·每章被写手读。**两条铁约束**(否则塌成 L3 洗稿):**指纹压文风参考**(你的 `写作指纹` 是主、范文是佐料,冲突时指纹赢);**只学节奏/措辞/句式、不许搬专名或整句原文**(镜像拆书的专名黑名单)。它**不改写作指纹的学习信号**(仍只从你的改稿学),故是每章助推、非学习源,"越写越像你"仍成立(详 [ADR 0012](docs/adr/0012-reference-others-influence-voice.md))。
_Avoid_: 让它盖过写作指纹(违「指纹压范文」);把整句原文搬进正文(违护栏=洗稿);把它当学习增量回写进指纹(它只在写章 prompt 里当样例,绝不进 learn);和 `拆书` 混为一谈(一个喂 voice、一个去 voice)

**审稿留痕**:
编辑就地改好之外、用哨兵 `<!--LOOM:EDIT-NOTE-->` 分隔附出的《本章改动留痕》;落到盘外 `.审稿留痕/`,给作者看。落盘前从终稿与 `.原稿` 快照剥净,**绝不进 learn 的 diff 源**。目录里还有一份全书一份的 `领航员留痕.md`(文件名沿用旧名,守「领航员」词条命名公约):书房伙伴对话循环的**失败类现场记录**(`why` 枚举 `backend_error`/`tool_unparsed` 等,只收失败、不收正常对话),只追加、不读回派生状态,可删。
_Avoid_: 让它进正文/快照/写作指纹;把它当打分或阻断(它只是留痕,不打分不阻断)

**续跑账本(ledger)**:
`正文/.原稿/第N章.ledger.json`,记每道工序的 sha + 产物 + 上游签名,供断点续跑(跳过"已完成且上游未变"的工序)。纯工程账本,与设定(what)、文风(voice)都正交。
_Avoid_: 给它加 schema 版本协商/打分/投影(坏了就当无 ledger 重跑,刻意极简)

**质量关卡(gate)**:
挂在某一棒产出之后的**独立复审 → 回炉重写**循环(质检挂编辑后、去AI味挂润色师后)。复审员按 rubric 只挑**具体硬伤**(带原文证据)→ 有硬伤就针对性回炉、再审,最多 `[gate]"轮数"` 轮(`轮数`=复审次数、末轮只诊断:**`1`=默认=只圈错写进留痕、不自动改稿**,`≥2` 才自动回炉,`0`=关)。判据是**「硬伤清单空不空」,不是分数**;跑满仍有残留则**不阻断**,保留最好稿 + 把残留追加进审稿留痕(见 [ADR 0006](docs/adr/0006-quality-gates-issue-driven-non-blocking.md))。
_Avoid_: 给它打分/挂检测分数门槛(重蹈 ADR 0002 覆辙);硬阻断出稿(违"留痕不阻断");把关卡产物回流写作指纹

**确定性预筛(本地检测器)**:
复审除了 LLM,还可挂**本地零成本检测器**先出一批「硬伤 + 原文证据」并入复审——去AI味挂 `aitell.py`(抓**句内**「不是A而是B / 不是A,是B」类 AI 翻转句)+ `fatigue.py`(抓**跨章**章首章末雷同/整句复用,Dice 二元组相似度,看最近数章)。守同一口径:**不打分**、可驱动回炉(`轮数≥2`)、默认(`=1`)只进审稿留痕;命中**先过写作指纹护栏**(比对 anchor 例句豁免作者签名句/有意复用的母题句,默认不碰句号切断的并列否定),绝不把「像你」当 AI 腔擦掉。是 `去AI味`(擦通用机器味)的**确定性地板**,不是新的打分维度。
_Avoid_: 给它设"命中数阈值/必须归零才放行"(变相打分硬闸,违 ADR-0002/0006);把它当过检测器作弊;不过指纹护栏就直接擦掉作者的不规整/口头禅

**伏笔悬空提醒**:
写第 N 章时(编辑棒后)扫卡章纲里 recap 自动记的 `[埋设]/[推进]/[回收]` 伏笔行,把"埋了超过 `[gate]"伏笔提醒章距"` 章、后文却查不到任何推进/回收"的伏笔追加进**审稿留痕**提醒作者。纯本地字符串匹配(剔除主角名等高频噪声字后比对区别性词),**不打分、不回炉、不阻断**——只**回头诊断**("你第K章埋了X、一直没还"),绝不替作者写前向规划或自动兑现伏笔。卡章纲没跑过 learn(无 `[AI回顾]` 伏笔行)时自然为空;`伏笔提醒章距=0` 关。借 inkos「未回收伏笔按章距升级」的思路(仅思想、未取其 AGPL 代码),只取这一内核。
_Avoid_: 给它建实体库/依赖图/半衰期/打分(inkos 那套重机械,违"简单是特性");让它进回炉或硬阻断;自动改卡章纲的前向规划行(向前规划永远归人,ADR-0007);因读卡章纲、把它塞进 gate 的 detector(质检回炉改正文清不掉它、只会空转——故走独立的留痕提醒路径)

**违禁词自检**:
对照 `外置大脑/违禁词.md`(国内平台常见雷区的分类指引 + 触发词样例),对正文做的**本地子串粗筛**;命中只在界面/审稿留痕里**提示、绝不阻断**,不发任何 LLM。是外置大脑里一份**人可增删**的清单。
_Avoid_: 当成"过审保证"(只兜低级漏网,终审靠人);堆砌敏感词原文(只放代表词 + 改写指引)

**版本历史 / 回收站(防丢稿)**:
所有写盘走**原子写**(写 `.tmp`→fsync→`os.replace`,断电/崩溃不留半截);正文每次被覆盖前先把旧稿快照进 `正文/.历史/第N章/<时间戳>.md`(留最近若干份,可一键回滚),删章则整章产物移进 `正文/.回收站/`。
_Avoid_: 和「整本 zip 备份」混为一谈(备份是灾备,历史是单章后悔药)

**章节管理**:
对已有章的删 / 插空章 / 上下移;引擎做**安全重编号**——把该章全部关联产物(正文 / .原稿 / ledger / .历史 / .审稿留痕 / .指纹历史 + `state.learned` 章号)两段式一起搬。
_Avoid_: 自动改卡章纲(它是人写的规划,只提示作者同步章号)

**每章标题**:
正文文件**首行 `# 标题`** 作单一真相(写完一章 AI 自动起、附赠失败回退无标题;可手改首行改名)。标题**只存名字、不含「第N章」**(章号由文件名带,重编号不动标题)。**红线:标题绝不进 learn 的文风学习**——标题进正文文件且 `.原稿`快照/`ledger` 同口径带上它(保 drifted/局部重写口径一致),只在量「人手改了多少正文」的三处(learn 的 diff、`chapter_drifted`、「改过」徽标)先 `strip_title` 再比。详见 [ADR 0009](docs/adr/0009-model-routing-write-safety-chapter-titles.md)。
_Avoid_: 把「第N章」写进标题内容(重编号后会陈旧);拿标题单独存 state/独立文件(多一套要同步搬的真相)

**模型路由(多供应商)**:
`backends.py` 的 `PROVIDERS` 注册表作路由唯一真相;四个一等公民:`deepseek`(自带 key、锁死 base_url)、`claude` / `codex`(复用客户端登录免 key)、`openai_compat`(**自填 base_url** 的通用口子,接智谱GLM/Moonshot/Qwen/硅基流动等)。模型框是**可编辑下拉 + 「拉取可用模型」**(实时 `GET /models`),不写死会过时的白名单。默认模型 `deepseek-v4-pro`(更强;更快更省可选 `deepseek-v4-flash`)。**DeepSeek V4 是思考型**:它先把推理写进 `reasoning_content`、答案才进 `content`,而推理也吃 `max_tokens`——故 `backends._budget_tokens` 为 DeepSeek 留思考余量(底线 6144、封顶 8192;别家供应商维持原样),否则短步骤(标题/复审)的预算被思考占满、正文空着回来报空响应(用户实报,已修)。可选 `cheap_model`:复审(质检/去AI味)+ 写后摘要/补设定这类**「评估、管 what」**调用走便宜模型(`cheap_backend()` 同 provider/base_url/key 换 model),**写作与学指纹始终用主模型保「像你」**;空=全用主模型。粗粒度一刀切,**不**做 inkos 的逐 agent 模型矩阵。
_Avoid_: 把模型名硬编码成固定白名单(厂商一改名就过时再炸);自建代理 / 卖订阅(见记忆「自带 key 决策」);把 `cheap_model` 铺到写作/学指纹上(省钱省到「像你」头上)

**写盘安全闸**:
`guard.py`,一切「模型输出 → 覆盖用户文件」的写盘点(指纹 learn/seed、正文终稿、流水线每棒、起草/补设定/摘要)落盘前校验(非空 + 最小实字 + 结构标记),不合格就**保留旧文件、抛友好错误**——根治用户实报的「换模型→learn 学到空→指纹被擦」。learn 分**硬闸**(空/残废→拦)与**软闸**(变短/丢 anchor→仍写 + 提示可撤销,守 ADR 0001 不自动打分)。
_Avoid_: 给指纹做严格质量硬闸(会误杀合法精简、等于自动打分,违 ADR 0002);硬阻断附赠类(enrich/recap 空响应只跳过)

## Relationships

- 一个 **agent** 通过它的 `reads:` 清单引用零或多个 **外置大脑** 文件和 **skills** 文件
- **外置大脑** 与 **skills** 的区别是**作用域**:前者每本书独有且会变,后者跨书复用且基本不变——而不是内容或格式
- 一条流水线由多个 **agents** 顺序组成;每个 agent 把产物追加进**本章工作区**,下游读**整个工作区**(累积,非纯链式)
- **领航员** 与流水线五工序互不伸手:领航员管旅程(问什么、何时问),产物只进外置大脑;流水线读到的访谈答案与人手写的别无二致(卡章纲行/世界观小节),不新建数据通道
- **外置大脑** 里 loom 会自动写的只有两类:**写作指纹**(整文件,从学习增量重蒸馏)+ **卡章纲/世界观/人物卡** 文末或行下**物理隔离的 AI 子块**(`[AI回顾]`/`[AI补充]`,只追加、绝不覆盖人写的主体);人手写的设定主体与大纲规划行永远由人维护
- **写作指纹** 的回写信号只能是 **学习增量**,不能是 agents 的输出
- **去AI味**(独立功能:润色师 + `去AI味.md`)负责擦掉**通用机器味**;**写作指纹**负责盖上**你的个人嗓音**——两件不同的事,不混为一谈
- "能不能过 AI 检测"是 去AI味 + 写作指纹 叠加后的**自然副作用**,**不是写作指纹的 KPI**,工具不量化、不上报(见 ADR 0002)
- **`loom write N`** 把 `正文/第(N-1)章.md`(**手改后的**那份)读进工作区做行文衔接;跨章记忆靠**此 + 卡章纲(含 learn 自动补的写后摘要)**,不做向量检索式前情召回
- **立项卡** 只被两处读:`平台` 字段 → 违禁词自检的基线松紧(粗粒度严/默认二档,仅在用内置基线时叠严格档);整卡 → 设定师的定位背景。loom **从不自动写它**(同 `违禁词.md`——外置大脑里人手维护、只读不回写的那类);对标素材本身**不入卡**(另贴给 `拆书`)
- **参考别人的作品**分两条对立通道,永不合流:**what 侧**走 `拆书`→挑进 `世界观.md`(去 voice);**voice 侧**走 `文风参考`→`写手` reads(留 prose,受「指纹压范文」+ 专名/原句护栏约束)。两条都**不回流写作指纹**——voice 的学习信号永远只是你的改稿(阀①,守 [ADR 0002](docs/adr/0002-fingerprint-purpose-and-division.md)/[ADR 0012](docs/adr/0012-reference-others-influence-voice.md))
- **境界/金手指/地名等硬设定 + 人物专名**经 `_hardfacts_for` 确定性切片后**逐字**进大纲师/写手,绕开设定师锚点的有损复述——专名/等级跨章不漂靠这条;**冰山真相**则 deny 不直送(防剧透)

## Flagged ambiguities

- "世界观" 曾同时指三样东西:`外置大脑/世界观.md`(数据)、`skills/世界观引擎.md`(方法论)、`agents/世界观.md`(工序)。**已解决**:agents 改用角色名(设定师/大纲师/写手/编辑/润色师),让三层永不共享同一个名词。
- "写作指纹的目的" 曾试图绑定"防 AI 检测/检测器分数"。**已解决(撤回)**:指纹目的回到纯"像你";"不像 AI"是 **去AI味**这个独立功能的职责,过检测只是副作用、不当 KPI(见 ADR 0002)。
- "skill" 有两副面孔:`/Desktop/网文skills` 里的**源文件**(250–340 行 Claude-skill,带 frontmatter/斜杠命令/评估模式)vs 项目 `skills/*.md` 里的**蒸馏版上下文**(只留写作硬核,喂进 prompt)。**已解决**:源是源,项目里只放蒸馏版;初版手工蒸馏一次内置。
- "去AI味是反AI规则唯一持有者、从别处全抠走"曾想一刀切。**已修正**:`去AI味.md` 只持有**去机器味的擦除规则**(套话/排比/黑名单词);而**感官写作、show-don't-tell 是"写得好"的手艺,留在写手的 `网文大神.md`**——两者不是一回事,不强行抠走。
- "写手 voice 侧只准读 `写作指纹`+`网文大神`"曾是硬规(防污染)。**已放宽(L2)**:新增一个**受控例外** `文风参考`——由作者主动选择、且被三重约束(**阀②指纹压范文** + **专名/原句护栏** + **阀①不回流指纹**),是有意的、可收敛回你的借力,不是无意污染。对比:`题材套路` **仍禁** reads 进写手(那是无意的 what 污染,无三重约束)。详 [ADR 0012](docs/adr/0012-reference-others-influence-voice.md)。
- "分卷大纲 / 卷" 曾想作独立一层。**已解决(取扁平)**:Loom **无卷/arc 概念**;"卷"就是 `卡章纲.md` 里人写的 `## 第N卷·xxx` 分节标题 + 一句 arc 意图,底下照旧一章一行——大纲师读**整份**卡章纲故自动看见大弧,跨卷连续性靠 `[AI回顾]` 伏笔三态 + 伏笔悬空提醒,不靠卷层级。要工具"按卷做事"(卷级质检/卷末大回顾/按卷编号)才需真加一层,**当前刻意不加**。
