# loom init 离线铺脚手架,联网工作拆成独立命令

`loom init` 只负责**离线**铺项目骨架(永不联网、永不因缺 key/断网失败),`写作指纹.md` 先落一份中性默认指纹。所有需要 backend 的工作拆成独立命令:`loom seed`(从样本/继承提炼初版指纹)、`loom write <N>`(跑链出稿)、`loom learn <N>`(学改稿 diff 重蒸馏指纹)。命令面 = **init / seed / write / learn** 四条。

这么切是因为 `loom init` 是录屏里"铁定成功"的一步(文件树长出来,见发布计划对 §6 录屏可靠性的要求),不能背一个会翻车的联网副作用;且每条联网命令单独失败时互不污染。

## Considered Options

- **一条 `loom init` 既铺文件又顺手联网播种**(命令更少)——否决:init 会因缺 key/断网当场失败,违背它"最稳一步"的人设,与录屏可靠性直接冲突。
- **init 离线 + seed/write/learn 各自独立(选中)**——init 永不翻车,联网失败被隔离在单条命令内。
