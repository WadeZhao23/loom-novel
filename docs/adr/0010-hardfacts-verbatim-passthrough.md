# 硬设定逐字旁路:境界/专名绕开设定师复述直送大纲师/写手

起因是用户实报:写第 2 章时世界观里的**境界等级写错**(力量体系是字母级 F~SSS,正文却凭空冒出"一阶0级"这种没立过的子阶),**地名/人物专名前后漂移**(同一所"湘城一中"被写成"二中")。

根因在流水线的信息流:真正落字的**写手** `reads` 只有写作指纹 + 网文大神,**从不直接读世界观**。境界阶梯、金手指代价、地名这些硬事实,要先被**设定师**压成 ≤350 字「设定锚点」、再被**大纲师**写进细纲,才二手传到写手——经两道 LLM **复述**,专名和等级名就漂了。设定师的锚点是**语义选择**(这章该用哪几条规则),本就该意译;问题出在"必须逐字不变的硬事实"也走了这条有损通道。

## 决定:在语义锚点之外,加一条确定性的"硬设定逐字旁路"

`agents._hardfacts_for(project_root)`:纯字符串切片,把**必须逐字照搬**的硬设定从外置大脑原样取出,拼成一个「## 硬设定(逐字照搬…)」块,直送**大纲师/写手**的 prompt(设定师本就整本读世界观、编辑闸也复读,故不重复注入):

- **世界观**按二级标题关键词命中切整段原文:力量体系(境界阶梯)/ 金手指(代价上限)/ 地理·势力(地名阵营)。`_HARDFACT_KW` 是 allow-list。
- **人物卡**取「## 类型 · 名字」式标题作专名册——只认带 `·` 分隔符的标题,滤掉「## 主角」这类没填名的占位、以及 learn 追加的「## AI 补充…」段名。
- 设定师的 ≤350 字锚点**保留不动**:它继续管"这章用哪几条 + 情绪/处境"的语义选择;硬设定旁路只补"专名/等级一字不改"的保真。两者互补,不是替代。

## 红线(本功能的几条硬约束)

- **冰山真相绝不逐字喂写手**:终极反转逐字进 prompt 等于让写手提前抖包袱。`_SPOILER_KW`(真相/反转/底牌/终局…)deny **压过** `_HARDFACT_KW` allow——即便反转段标题里带"势力"也不放行;且连**嵌进被命中段的 `### 反转`** 子块也一并剥掉。这是本功能唯一不可破的红线。
- **纯确定性、不阻断**:只字符串切片,不调 LLM、不打分、不建实体库(守 [ADR 0002](0002-fingerprint-purpose-and-division.md)/[ADR 0006](0006-quality-gates-issue-driven-non-blocking.md));文件缺失/是目录/为空一律返回空串(`.is_file()` 守门),绝不阻断出稿——是附加增强,不是新关卡。
- **不进续跑账本签名**:硬设定只取自 世界观.md / 人物卡.md,而这两份都在**设定师**(永远 PIPELINE[0])的 reads 里——改硬设定必先让设定师签名失配、从第 0 棒全量重跑,大纲师/写手自然吃到新值。再折进 `_sig` 只会白白冲掉在跑章节的 ledger、触发无谓重计费(守"断点续跑省 key")。唯一缺口:纯代码改了切片逻辑而文件没动时,半截章续跑沿用旧切法——升级期一次性、自愈(`--force` 或动一下世界观即重算)。

## 与既有不变量的兼容

- **不动 learn/指纹/关卡**:旁路只读外置大脑、只进写章 prompt,不回写任何文件,不碰 learn 的 diff、不喂写作指纹(管 voice 的仍只有指纹),关卡照旧。
- **learn 自动追加的 `[AI补充]` 新层级不进逐字块**:它们挂在标题不含硬设定关键词的「## AI 补充」段下(且自动追加块不该被当真相直喂)。要让学到的新境界逐字护身,把它手并进 `## 力量体系` 正式段(向前规划/正式设定仍归人,守 [ADR 0007](0007-external-brain-grows-with-story.md))。
- **regen_outline 同口径**:「重新生成细纲」(设定师→大纲师)也吃同一份硬设定。

## Considered Options(选摘)

- **把整本世界观 reads 进写手**——否决:稀释写作指纹优先的落字(写手只读指纹+网文大神是刻意的),且会把冰山真相一起喂进去剧透。选"只逐字切硬设定小节 + deny 反转"。
- **靠 LLM 抽取硬设定 / 给一致性打分**——否决:违 ADR 0002/0006 的"不打分、不建库";确定性切片足够,且零成本零误伤。
- **把硬设定折进续跑签名**——否决:冗余(源文件已在设定师 reads 里)且会无谓毁 ledger 重计费;改注释讲清不变量即可。
- **人物专名册也从「- 名字:」小节里抽**——暂缓:默认模板的名字字段多是空占位或"名字/立场/功能"合并字段,抽出来不干净;先只认带 `·` 的标题(宁可空,不把段名当人名喂写手),够用再扩。
