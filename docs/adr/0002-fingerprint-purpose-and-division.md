# 写作指纹的目的是"像你"(个人嗓音);去AI味是独立功能,过检测是副作用而非指纹 KPI

写作指纹这条线的目的只有一个:**让生成的文字带上你的个人嗓音**(句式/用词/节奏/口头禅/禁用词,外加若干条逐字保留的你的原句作为 anchor 例句)。它的成功标准是产品**内部可判**的——"生成稿有没有体现你的偏好"(看 learn 的 diff、看对照样本即可),不依赖任何外部黑盒。

"不读着像 AI" 是另一个真需求,由**去AI味**这个**独立功能**(润色师 + `去AI味.md`)负责:擦掉通用机器味(套话头尾、空洞排比、AI 黑名单词)。它**不挂检测分数 KPI、不针对任何具体检测器作弊**,口径与 `去AI味/SKILL.md` 自带的免责声明一致("写得自然本就是降 AI 率最稳的办法")。两个功能组合:去AI味擦净机器味 + 写作指纹盖上你的嗓音;"能不能过某个检测器"是叠加后的**自然副作用**,工具不优化、也不上报这个数字。

## 为什么撤回"过检测当目的"(本 ADR 早期口径)

对抗审计指出:检测器测的是文本统计特征(困惑度/burstiness),**没有"像不像你"这一维**。把检测分数派给一个**个性化**模块(指纹)当 KPI,等于让指纹为一个只有去AI味搬得动的指标负责——产品因此失去"指纹到底有没有用"的自检能力,还让仓库内 `去AI味.md` 的免责声明与架构自相矛盾。故撤回,并放弃"先擦后盖"那个为绑检测而生的说法。

## Considered Options

- **过检测当目的、检测分数当成功标准**(早期口径)——否决:因果错位(指纹背了去AI味的 KPI)、不可自检、与自带免责声明矛盾。
- **指纹=像你、去AI味=独立反AI功能、过检测降为不量化的副作用**(选中)——自洽、可内部自检、不和自己的文件打架。
