# learn 的学习信号:从行级 diff 换成句级结构对齐 + 模型两步判定

`fingerprint.learn()` 喂给模型的"手改信号",从 `difflib.unified_diff(splitlines())`(行级)改成**句级结构对齐**:先把改稿切句,用 `SequenceMatcher` 把改动分成「改写候选」(同位置的 replace,= 文风候选)与「纯增删」(insert/delete,多半改剧情);再让模型**两步走**——第一步逐对判定「改写(换说法,文风)/ 改情节(发生的事变了)」,第二步只从「改写」里学文风,纯增删与改情节一律忽略。`_LEARN_SYSTEM` 同时要求 anchor 例句须是**纯文风、不含剧情专名/信息**。

## 为什么(这是写作指纹的命门)

写作指纹 = 像你(ADR 0002),`learn` 是 Loom vs「直接用 DeepSeek」的唯一差异点。但旧的行级 diff 有致命噪声:

- 网文作者改稿常是**整段重写** → 行级 unified_diff 变成「一大坨 `-` + 一大坨 `+`」,**没有句对**,模型看不到"AI 这句 → 你改成那句"的对应,只能从一团 diff 里瞎猜文风。
- 后果(A/B 实测,见下):混合改稿里,模型会把**新增的剧情事实焊进 anchor 文风例句**(例:把"这笔账三年了"当成一条文风模板),此后每章复现这条不该复现的信息——慢性污染"像你"。

## 一个绕不过的约束

要把"改写(文风)"和"改情节(剧情)"彻底分开,本质需要**语义相似度**(同义不同词 = 文风;异义 = 剧情)。而 Loom 的红线明确**不引入向量库/embedding/外部服务**(本地、极简、一个 key 走天下)。所以:

> **机械层只能按位置对齐,做不到语义切分;语义判定交给模型。**

句级对齐(stdlib `SequenceMatcher`,零依赖)负责把信号**结构化成干净的候选句对**并分离纯增删;"这对到底是不是文风改写"这个语义判断,由模型在第一步做——模型恰好擅长这件事,而旧的一团 diff 根本不给它做判断的结构。

## 证据(A/B,真实 LLM)

同一手改,旧法(行级 diff + 旧提示词)vs 新法(句级对齐 + 两步提示词),盲审判泄漏:

- **纯改剧情**:新旧都学到 0 条(明显纯剧情,模型本就不上当)——打平。
- **混合改稿(改文风 + 加新剧情)**:旧法 6 条里 **1 条把"这笔账三年了"当文风**泄漏;新法 5 条**全纯文风、零泄漏**。盲审判新法胜。

即新法 ≥ 旧法,在最常见的"混合改稿"上更干净,且随章节变长(行级 diff 退化最狠处)收益更大。

## Considered Options

- **行级 unified_diff(旧,否决)**:整段重写时无句对、易把剧情焊进 anchor。
- **embedding 语义对齐(否决)**:能真正按语义分文风/剧情,但要向量库/外部 embedding 服务,**违反 Loom 本地·极简·无向量红线**;且把 Loom 推向重基础设施。
- **让用户在编辑器圈选「这是嗓音改动」(暂缓)**:信号最干净,但有交互摩擦;留作日后可叠加的补充,不作为本次主路径。
- **句级对齐 + 模型两步判定(选中)**:零新依赖、合红线;机械层给干净句对、模型做语义判定,实测降低剧情泄漏。

## 影响 / 局限

- `_diff` 删除,新增 `_segment` / `_aligned_signal`;`_LEARN_SYSTEM` 改两步走 + anchor 纯净约束。其余(seed、reads、recap 挂钩)不变。
- 句切按 `。!?…\n` 等,引号内句末标点等边角不完美,但对齐质量足够;后续可细化。
- **仍需真稿验证**:本 ADR 证明的是"信号更干净、剧情泄漏更少";"越写越像你是否肉眼可见"仍要在 2-3 章真稿 + 真实后端上压测(见 [[learn-loop-line-diff-risk]] 的收口条件)。这是无基础设施前提下能做到的最优解,不是终点。
