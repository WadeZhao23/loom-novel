# 外置大脑随章生长:learn 后向世界观/人物卡【追加】物理隔离的 [AI补充],绝不覆盖人手写的

把"外置大脑会随故事生长"(CONTEXT)从口号落成机制。在已有的卡章纲 `[AI回顾]`(向后的写后摘要)之外,再让 **世界观.md / 人物卡.md** 也随写随长:`loom learn` 接受一章后,从**手改定稿**里蒸馏出**这章新冒出来、且现有文件里还没记下**的世界观设定 / 人物信息,以 `[AI补充·第N章]` 块 **write-once 追加**到这两个文件末尾的「AI 补充」区。

这回答的是作者的真实疑问:"世界观、人物卡、大纲后面只能人为修改吗?"——不。它们能像写作指纹一样随故事长,但**人始终拥有最终所有权**。

## 红线(与 recap/[AI回顾] 同源)

- **只读手改定稿**(`正文/第N章.md`),不是 `.原稿` 快照;只学已经写成的事。
- **只管 what**(设定/人物),**绝不喂写作指纹**(voice 只能由学习增量更新,见 [ADR 0001](0001-living-writing-fingerprint.md)/[0002](0002-fingerprint-purpose-and-division.md))。
- **只追加、绝不覆盖**:loom 只在文件末尾的「AI 补充」区追加 `[AI补充·第N章]` 块,**逐字保留作者手写的世界观/人物卡主体**;作者可改可删这些 AI 块。
- **write-once**:同章重复 learn 不重复追加(人写优先);两边都已补过这章就连 LLM 都不调。
- **不引入实体库/打分/向量**:就是一次 `backend.complete` + 字符串追加,刻意极简。

## 大纲(向前规划)仍归人

只让**向后的**回顾/补充自动长(回顾=这章写成了什么,补充=这章确立了什么)。**向前的**大纲规划行不自动写——预测"未来该写什么"易跑偏、且和作者手写的剧情脊柱抢方向。需要时作者可让大纲师**按需草拟**一版来挑,而不是自动落盘。所以卡章纲的人写规划行永远由人维护,只在其下自动补 `[AI回顾]`。

## 与既有不变量的兼容

- **附赠、绝不阻断 learn**:enrich 跑在 `fingerprint.learn` 里、`mark_learned` 与指纹落盘**之后**,且整段 `try/except`——任何失败(后端报错/解析异常)都只发一条 warn,指纹已经稳稳落盘。
- **喂回流水线天然成立**:设定师本就 `reads: 世界观.md / 人物卡.md`,追加的 `[AI补充]` 是这两个文件的一部分,下一章设定师**自动读到**,跨章设定连续性更强——无需新增 reads 或工序,PIPELINE 仍是 5 棒。
- **删章联动**:`delete_chapter` 在把整章产物移进回收站时,连同卡章纲的 `[AI回顾]` 一起,把世界观/人物卡里本章的 `[AI补充]` 块也 strip 掉并**留底回收站**。否则同号重新生成、再 learn 时会被 write-once 挡住,外置大脑里留着的还是【已删旧章】的设定(本次同时修掉的一个真 bug)。
- **后端协议不变**:只用 `backend.complete(system, user)`,DeepSeek/claude/codex 同样能跑。

## Considered Options

- **每章把设定师/大纲师的临时产物自动回存到外置大脑**——否决:那是"本章锚点/分镜"的工作区中间态、含大量临时推断,直接覆盖会把作者手写的设定真相冲掉,违"外置大脑人主导"。
- **让 AI 就地改写世界观/人物卡正文**(精准插进对应小节)——否决:太脆、易误改人写内容;改用"文末追加物理隔离块",和 `[AI回顾]` 一致、可一眼分辨、可整块删。
- **文末追加 write-once `[AI补充]` 块、删章联动清理**(选中)——只追加不覆盖、可证据化、与 ledger/后端协议/写作指纹全兼容,且把外置大脑的"生长"做成人可随时接管的。

## 已知取舍

几十章后世界观/人物卡会累积许多 `[AI补充]` 块,设定师每章读全文→上下文渐长(与卡章纲 `[AI回顾]` 同性质)。v1 接受;日后若需要,可加"合并旧补充为一段"的压缩(类比写作指纹的 40 条上限),不改本 ADR 的追加语义。
