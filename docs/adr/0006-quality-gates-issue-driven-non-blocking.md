# 质量关卡(gate)挑硬伤、不打分、不硬阻断:复审→回炉循环挂在编辑/润色师之后

给写作流水线装"手脚"的第一步:把原本**一次性过一遍**的质检(编辑读 `评估自检.md`)和去AI味(润色师读 `去AI味.md`),升级成**独立复审 → 回炉重写**的关卡。每道关卡:一个**独立复审员**按 rubric 只挑**具体硬伤**(带原文证据)→ 有硬伤就**针对性回炉重写**、再复审,如此最多 `gate.轮数` 轮。注意 **`轮数` 数的是「复审」次数、且末轮只诊断不回炉**,故 `轮数=N` 最多回炉 N−1 次——**默认 `轮数=1` 即「只复审一遍、列出硬伤、不自动重写」**(残留入留痕、由作者定夺);要它自动回炉重写,显式调到 `轮数≥2`。

关卡只判**「硬伤清单空不空」**,不产任何分数:
- **质检**(挂编辑之后):只挑 人物OOC / 设定漂移 / 断钩子 / 整章无爽点——读人物卡/世界观/卡章纲/上一章作判据。
- **去AI味**(挂润色师之后):只挑 `去AI味.md` 黑名单的**具体命中词句**,并以写作指纹为护栏(个人特征不算 AI 腔)。

跑满轮数仍有残留 → **不硬阻断**:保留当前最好稿,把残留**追加进 `.审稿留痕/`** 交作者定夺,流水线照常往下。

## 为什么不打分、不阻断

直接沿用 [ADR-0002](0002-fingerprint-purpose-and-division.md) 的结论与 CONTEXT「审稿留痕不打分不阻断」:检测/质量**分数**当 KPI 会造成因果错位、不可自检、与仓库内免责声明自相矛盾。所以关卡用**问题驱动**(issue-driven)而非**分数驱动**(score-driven)——它回答"这章有没有具体硬伤"这个可证据化、可自检的是非题,而不是"打几分"。"过不过 AI 检测"依旧只是去AI味+写作指纹叠加后的副作用,工具不量化、不上报。

不硬阻断则守住"留痕不阻断":作者始终拿到一份完整终稿 + 一份"未解决项"清单,而不是被一个自动判罚卡住出不了稿。

## 与既有不变量的兼容

- **ledger/续跑**:关卡跑在某一棒的 `record_step` 之前,记录的是**过关后**的产物;rubric 文件已在该棒 `reads:` 内,改了 rubric 上游签名变化→自动重算。关卡是该棒内部的子步骤,不新增流水线工序,PIPELINE 仍是 5 棒。
- **后端协议不变**:关卡只用 `backend.complete(system, user)`(复审一次 + 回炉≤N 次),DeepSeek 等纯文本后端照样能用——属"引擎编排工具"路线,不要求后端支持 tool calling。
- **写作指纹**:去AI味关卡显式以指纹为护栏,绝不把个人特征当 AI 腔擦平;关卡产物**不回流写作指纹**(仍只有 `loom learn` 的人改 diff 能更新指纹)。

## Considered Options

- **分数门槛关卡**(打分<阈值就卡住)——否决:重蹈 ADR-0002 撤回的覆辙,因果错位、不可自检、与免责声明矛盾。
- **问题驱动 + 跑满不阻断、残留入留痕**(选中)——可证据化、可自检、守住"留痕不阻断",且与 ledger/后端协议/写作指纹全兼容。
- **放开后端自带工具让其边写边查**(B 路线)——暂缓:只限 agentic 后端、不确定性高、续跑与成本难控;待本路线跑稳后再评估。

## 开关

`loom.toml` 的 `[gate] "轮数"`:

- **`1`(默认)= 只诊断**:复审一遍、把硬伤写进 `.审稿留痕/`,**不自动改稿**。
- **`≥2` = 自动回炉重写**:`轮数=N` 最多回炉 N−1 次(末轮只复审、不再回炉)。
- **`0` = 关闭**整道关卡。

**为什么默认只诊断、不替作者改**:守 CONTEXT「不硬阻断、把决定权还给作者」——默认只圈硬伤、留给作者定夺要不要照改;想要开箱即用的自动回炉,再显式开到 `2`。这也更省:回炉=每道 gate 多一次整章重写调用(DeepSeek 廉价、claude/codex 明显变慢)。
