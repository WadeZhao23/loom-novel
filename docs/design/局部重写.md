# 功能设计 · 局部重写(子章节迭代)

## 要解决什么

现在的模型是「整章生成 → 文本框手改」。作者真正要的是**一段一段地改**:选中一段 → 让它按你的嗓音重写 / 按指令调整 → 用哪个你定。这是把 Loom 从「初稿生成器」推到「作者愿意天天泡在里面的写作工具」的那一刀;且每次重写 + 你的取舍,都是新的 learn 信号。

## 用户流程

1. 在编辑器里打开一章,**选中**要改的一段(一句/一场)。
2. 点「✂️ 重写选中」→ 弹层显示选中段 + 一个**指令框**(可空,如「太铺张,压一压」「换个更冷的章末钩」「这里加一句冲突」)。
3. 点「重写」→ AI 按**写作指纹的嗓音** + 指令,只重写这一段(整章作上下文)→ 候选显示在弹层。
4. 作者选:**「用这个」/「再来一个」/「取消」**。
5. 「用这个」→ 把选中段替换成重写段,落盘。

## 铁律(本功能的命脉)

局部重写产出的是 **AI 文本**。而 ADR 0001 的铁律是:**learn 只学你的手改,绝不学 AI 输出**。learn 的对照基线是 `正文/.原稿/第N章.md`(AI 终稿快照)。

所以:**应用一次局部重写时,必须把这次 AI 改动【同样落进快照】**,否则下次 learn 会把这段 AI 重写当成「你的改动」学进指纹 → 污染「像你」。

- **人手改** → 只写 `正文/第N章.md`(快照不动)。← 现状,正确。
- **AI 局部重写** → 写 `正文/第N章.md` + **外科式更新快照**:把快照里的旧段替换成新段(只换这一处),其余不动。

**外科式更新(关键)**:`snapshot.replace(旧段, 新段, 1)`。
- 常见情形(改的段 ≠ 之前手改过的段):旧段仍在快照里 → 精准替换。于是快照 = (你手改过的部分仍是原 AI 版)+(这段的新 AI 版),learn 仍只看见你手改的那部分。✓
- 退化情形(对你刚手改过的同一段再做 AI 重写):旧段已不在快照 → 退回「整章作新 AI 基线」(把当前正文整体写进快照)。代价:这次之前的手改被并进基线、不再算 learn 信号。**少见,且安全降级(只丢信号,不污染)**。

> 一句话不变量:**快照永远 = "若全程没有人手打字、只有 AI 生成" 的那一版**。AI 操作(write / 局部重写)更新它;人手打字不更新它。

## 落地

### 新增 `loom/rewrite.py`
```python
_REWRITE_SYSTEM = """你是作者本人的写手。我给你整章正文作上下文、作者选中要重写的一段、以及作者的指令。
按【写作指纹】的嗓音重写【选中段】,满足指令。只输出重写后的那一段本身,不要整章、不要解释。
默认不改剧情/人物/设定(除非指令明确要求);长度与原段相当;与前后文衔接自然。"""

def rewrite_span(project_root, chapter_n, full_text, span, instruction, backend, progress=_noop) -> str:
    fp = (project_root/"外置大脑"/"写作指纹.md").read_text()  # 读嗓音
    user = f"## 写作指纹\n{fp}\n\n## 整章上下文\n{full_text}\n\n## 选中段\n{span}\n\n## 指令\n{instruction or '(按你的嗓音改得更顺,别改剧情)'}"
    return backend.complete(_REWRITE_SYSTEM, user, max_chars=len(span)+300).strip()

def apply_rewrite(project_root, chapter_n, new_content, old_span, new_span):
    out  = project_root/"正文"/f"第{chapter_n}章.md"
    snap = project_root/"正文"/".原稿"/f"第{chapter_n}章.md"
    out.write_text(new_content.rstrip()+"\n", encoding="utf-8")
    if snap.exists():
        s = snap.read_text(encoding="utf-8")
        s2 = s.replace(old_span, new_span, 1) if old_span and old_span in s else new_content.rstrip()+"\n"
        snap.write_text(s2, encoding="utf-8")
```

### server.py
- `POST /api/rewrite` {root, chapter, full_text, span, instruction} → `{rewritten}`(只生成,不落盘)。
- `POST /api/rewrite/apply` {root, chapter, content, old_span, new_span} → 落盘 + 外科式更新快照,返回 `{ok}`。

### WebUI(编辑器)
- 章节文件打开时,编辑器动作区多一个「✂️ 重写选中」。
- 弹层:选中段(只读)+ 指令框 + [重写];出候选后 [用这个]/[再来一个]/[取消]。
- 「用这个」:用 textarea 的 selectionStart/End 把选中段替换成重写段 → 调 /api/rewrite/apply → refresh。
- 客户端持有 old_span(原选中文本)和 new_span(重写文本)一并发给 apply,供外科式更新快照。

## 验收标准
- [ ] 选中一段 → 重写 → 用这个:正文里该段被替换,其余不变。
- [ ] **铁律测试**:write → 手改 A 段 → 对 B 段做 AI 局部重写 → learn:learn 只学到 A 的手改,**不把 B 的 AI 重写当文风学**(快照外科式更新生效)。
- [ ] 退化情形:对刚手改过的段再 AI 重写 → 不污染指纹(退回整章基线,只丢信号)。
- [ ] 空选中 → 友好提示;rewrite 读写作指纹,产出带作者嗓音。
- [ ] 局部重写不进 learn 触发、不碰卡章纲/写后摘要;纯本地。

## 不做(v1 克制)
- 不做多候选并列(先 1 个 + 「再来一个」);不做流式(段短,一次返回);不读外置大脑(整章上下文够,日后设定漂移再加);不做跨段批量重写。
