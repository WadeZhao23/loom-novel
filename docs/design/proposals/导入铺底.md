# 导入铺底 —— 资料夹一键接成一本新书（设计定稿 · 2026-07-07）

**问题**：用户（陈二筒等）已用别的 AI skill 攒了大量**非-Loom 的散装 md**（世界观/人物/大纲/章纲），手里有完整大纲章纲、只差正文。想接进 Loom 直接进写作流水线,但建新书还得对着**空表格重填**,且他的 md 结构/分类与 Loom 外置大脑(世界观/人物/卡章纲…)对不上。他还点评「那些 skill 强在设定、弱在写作」——写作正是 Loom 的强项,只差一条低门槛的接入路。

## 决策记录（grill-with-docs 拷问定案,2026-07-07;词表见 CONTEXT.md「导入铺底」）

| 决策点 | 结论 | 拷问要点 |
|---|---|---|
| 接不接散装 md | **接**:资料夹 → 一本新书 | 散装 md 灌进外置大脑**不会损坏结构**(引擎防御式:缺文件/畸形静默降级,唯一硬要求 loom.toml 由我们生成);风险是「静默不生效」不是「坏」 |
| 机械 vs AI 重塑 | **纯机械搬运**,不调 LLM、不重塑他的内容 | 守「loom 不发明设定、不改写人写主体」;用户实报最怕 AI 改坏他的料 |
| 谁分类落位 | **文件名启发** + **猜不中让用户指认** | 只在不确定处一次交互,守从简 |
| AI 归位(重塑成 Loom 结构) | **不进本期** | 有失真风险;硬设定不命中靠**导入小结明示降级** + 事后二期「世界观 AI 改写」手动修 |
| 目标 | 资料夹 → **新**书(v1);合并进已有书不做 | 合并的去重/覆盖语义另说 |
| 版本 | 0.3.6 | — |

## 设计

### 1. 路由（`loom/importer.py`,纯函数）

扫资料夹里的 **`.md` 文件**（只吃 md,忽略图片/docx/txt）。按**文件名关键词**启发路由到桶:

| 文件名含 | 落桶 | 形态 |
|---|---|---|
| 大纲 / 章纲 / 剧情 / 分卷 / 卷纲 | `外置大脑/卡章纲.md` | 单文件,**多份拼接**(每份前加 `## 来自:<原名>` 分隔) |
| 人物 / 角色 / 主角 / 配角 / 反派 / 小传 / 人设 | `外置大脑/人物/` | 目录,一份一文件原样落 |
| 设定 / 世界 / 力量 / 体系 / 境界 / 地理 / 势力 / 金手指 | `外置大脑/世界观/` | 目录,一份一文件原样落 |
| 立项 / 定位 / 平台 | `外置大脑/立项卡.md` | 单文件(覆盖模板) |
| 违禁 / 敏感 / 审核 | `外置大脑/违禁词.md` | 单文件(覆盖模板) |
| 文风 / 范文 / 风格 | `外置大脑/文风参考.md` | 单文件(覆盖模板) |
| **命中不了** | → `unknown` 列表,交前端让作者指认 | — |

`route_files(names: list[str]) -> dict`:返回 `{"世界观": [...], "人物": [...], "卡章纲": [...], "立项卡": [...], "违禁词": [...], "文风参考": [...], "unknown": [...]}`。**不含写作指纹桶**——写作指纹是 learn 从手改蒸出的结构化文件,不接受粘贴原文(他若有文风素材走 seed/文风参考)。同名撞两类(如「人物大纲」)→ 落 unknown 让作者定。纯字符串,不调 LLM。

### 2. 落盘（`importer.import_folder`）

`import_folder(folder: Path, name: str, routing: dict, parent: Path) -> Path`:
1. 复用 `scaffold.init(name, parent)` 建标准骨架(loom.toml/模板/中性写作指纹/题材可后补);
2. 按 `routing`(前端确认后的最终映射,含作者对 unknown 的指认)把每份 md **原样**落进对应桶:目录桶(世界观/人物)一份一文件、文件名净化非法字符;单文件桶(卡章纲/立项卡/违禁词/文风参考)多份**拼接**、每份前加 `## 来自:<原名>` 溯源分隔;
3. 全程原子写(走 `atomic_write_text`),不改一个字。

书名默认取**文件夹名**(可在导入弹层改)——把「重填表格」变成「确认预填」。

### 3. 导入小结（`importer.import_summary`,降级可见）

`import_summary(root: Path, routing: dict) -> dict`:落盘后出一份**只读小结**(不调 LLM):
- **落位**:各桶收了几份、哪几份(给作者一眼看清都放哪了);
- **降级明示**(复用现成检测):
  - 跑 `agents._hardfacts_for` 看世界观有没有命中硬设定小节;零命中 → 提示「没识别到硬设定小节(力量体系/地理势力),境界专名可能漂——去世界观里选中用『AI 改写』把它单列成一节即可」(指路二期已有能力);
  - 卡章纲不是一行章纲格式 → 提示「时间轴/伏笔账本按『- 第N章:』格式挂,你的章纲是段落式、这些自动记忆暂不显示,不影响写正文」;
  - 无正文 → 提示「你还没有正文,写作先用中性文风,写几章 learn 后越来越像你(或用 seed 从范文起手)」。

### 4. server 端点

- `POST /api/project/import/scan` {folder} → `{name_suggest, routed: {桶:[文件名]}, unknown:[文件名]}`(扫描+启发路由,不落盘,供前端出确认屏)。
- `POST /api/project/import/commit` {folder, name, parent, routing} → 顶层即 project_state + `summary` 键(`{**state, "summary": summary}`,与 `create_project` 同扁平形状,`enterProject(d)` 直接吃 d.root/d.title)。
- 复用 `_shelve` 入书架;错误走 `_err_json`。folder 为作者原生文件夹选择器给的绝对路径,校验 `is_dir()`/含 `.md`(不适用 `safe_join`——那是约束 rel-within-root 的,源夹在项目外)。

### 5. webui

- 欢迎页「开一本书」区加**第三个入口**「📁 导入资料夹(从已有设定/大纲接一本新书)」,与「新建」「导入已有 Loom 书」并列。
- 流程:选文件夹(pywebview `pick_folder`,web 回退填路径)→ 调 scan → **确认屏**:书名可改 + 各桶落位预览 + **unknown 逐个指认**(下拉选桶或跳过)→ 调 commit → **导入小结弹层**(落位 + 降级提示)→ enterProject。
- unknown 指认复用 showGuide/弹层样式;确认屏用现成的 create-overlay 家族样式。

## 红线与明确不做

- **不调 LLM、不重塑内容、不 AI 填立项卡/违禁词**（只搬他现成的、没有就留模板）。
- **不做 AI 归位**(硬设定/自动记忆不自动挂,靠导入小结明示 + 二期世界观 AI 改写手动修)。
- **只 .md、只资料夹→新书**（合并进已有书、非 md 格式、AI 建议拆点均不做）。
- **写作指纹不是路由桶**（voice 走 seed/文风参考）。

## 影响面与测试

- 新增 `loom/importer.py`(route_files/import_folder/import_summary 三纯函数式接口)、server 两端点、webui 一入口+确认屏+小结弹层。复用 scaffold.init/atomic_write_text/_hardfacts_for/_shelve,零改引擎红线。
- 测试:route_files 启发命中/撞类落 unknown/写作指纹不在桶;import_folder 目录桶一份一文件+单文件桶拼接+文件名净化+原样不改字;import_summary 硬设定零命中/段落式章纲/无正文三类降级提示;端点 scan/commit(FakeBackend 不涉,纯机械)+ safe_join 越界拒;webui preview 冒烟(选夹→确认屏→unknown 指认→小结)。
- golden 零影响(不碰流水线 prompt)。老书零迁移(纯新增入口)。
