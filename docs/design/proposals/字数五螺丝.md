# 字数五螺丝 —— 确定性反馈环治字数飘忽（设计定稿 · 2026-07-08）

**问题**：用户实报「字数还是飘忽不定」。三期「字数链路」调查结论:头三章无代码级偏置(仅黄金开篇+爽点闭环的间接偏置);真因是几颗没拧的螺丝 + 模型天花板。飘忽主因:①编辑/润色拿「超了就压」授权却从没被告知原稿实测字数(LLM 自己数不准=授权无牙);②改字数后旧细纲被 WYSIWYG 静默沿用(旧场次+旧每场字数总和≈旧目标),写手照旧细纲写=新目标必然失效;③超长阈值 1.5x 与写手 ±20% 脱节,1.2-1.5x 区间静默放行;④细纲「约X字」标注无确定性校验;⑤系统提示词硬编码「3-6场景」与任务行结构闸打架。

## 决策记录（brainstorming 定案,2026-07-08）

| 五期范围 | 字数五螺丝(确定性反馈环);拆章工具=六期(独立子系统另立 spec) |

## 设计（螺丝 ①③④⑤⑥⑦,② 已做）

### 螺丝①·注入实测字数(最有牙,确定性零成本)

`run_pipeline` 在编辑/润色棒**前**,确定性算上游稿实测字数 `n = len(re.sub(r"\s+","", strip_title(上游稿)))`(与 `_flag_overlong`/evals `grade_length` 同口径),传进 `_length_hint`。编辑/润色任务行从「原稿明显超目标就顺手压回来」改为带实数:「**原稿实测 N 字,目标 T 字(超 X%);删冗余描写/重复信息/注水铺垫,压回目标量级,不动情节骨架,绝不扩写。**」超标才提压缩、达标不提(避免对合格稿瞎压)。`_length_hint(role, step_budget, chapter_target, actual_chars=0)` 加参数,run_pipeline 两处调用点(主循环编辑/润色棒)传实测值;regen_outline 不涉(它只跑设定师→大纲师)。

### 螺丝③·超长改主动

`_flag_overlong` 阈值 `target*1.5` → `target*1.25`(与写手 ±20% 对齐,堵 1.2-1.5x 静默区);命中除追加审稿留痕外,发一个**独立 `overlong` 事件**(events.overlong(chapter, chars, target),前端可专门提示「本章超长可能注水」,不只默默写文件)。**不做自动压缩回炉**。

### 螺丝④·细纲字数校验 + 螺丝⑤·陈旧细纲提醒(共用解析器)

新增确定性解析器 `_parse_scene_budgets(outline: str) -> list[int]`:抓细纲里「约X字」标注(正则 `约\s*(\d+)\s*字`)返回各场字数列表。
- **④**:大纲师产出细纲后(run_pipeline 大纲师棒 output 落盘处),校验:一场都没标(空列表) 或 `abs(sum - chapter_target) > chapter_target*0.3` → warn「细纲各场没标字数/总和(M字)与目标(T字)差得多,写手篇幅可能失控」(非阻断,进 progress+留痕)。
- **⑤**:WYSIWYG 沿用已有细纲时(细纲文件存在→旁路大纲师那条分支),解析其字数总和 `sum`,若 `sum>0 且 abs(sum - chapter_chars) > chapter_chars*0.3` → info 提示「这份细纲按旧字数(约M字)拆的、当前目标T字,建议『重新生成细纲』」。**这是改字数后字数不听话最常见的触发路径,当前完全静默**。

### 螺丝⑥·模板去硬编码

`大纲师.md`/`故事引擎.md` 的「3-6 个场景」改「**场次以本章任务给的字数预算为准**(短章 2-3 场、长章才 4-6 场,别用多场把写手篇幅撑爆)」,消除系统提示词(常赢)与任务行结构闸(`_scene_budget`:≤1500→2-3场/≤3000→3-4场/>3000→4-6场)的打架。

### 螺丝⑦·evals 严口径

evals 加一档 `len_tolerance=0.25` 的 case(或新增 case_03 严字数),让 ±20% 承诺有回归防护。现默认 tol=0.5、实配 0.6(松 2.5-3 倍),±20%~±50% 飘忽 eval 永远测不出。

## 诚实天花板

LLM 无法边生成边准确计数,思考型 v4 输出长度还随情节复杂度漂;螺丝全拧上后单章 ±25-30% 残余波动仍在——那是能力天花板,不是工程缺口。架构(结构闸+压缩授权+留痕+确定性反馈)方向对,缺的是把「数字数」从模型手里拿回代码手里,本期补上。
