# 立项即铺底 —— 第一章不再漂移（设计定稿 · 2026-07-06）

**问题**：新书直接「织第一章」时正文与书名/立意毫无关系（实测《重生之我是崇祯，拯救大明王朝》第一章生成出通用金手指爽文开局）。

## 根因（三个叠加）

1. **书名从不进流水线 prompt**。`agents._build_user_prompt` 首行只有「你要写的是第 N 章」；`cfg.title` 全项目仅 draft 起草与 UI 展示在用。设定师/大纲师/写手不知道这本书叫什么。
2. **占位模板被当真设定执行**。新书的世界观/、人物/ 目录不存在；`卡章纲.md` 模板预填的示例行「第1章：主角在最糟处境登场（濒死/被废/被冤），金手指初现」被大纲师照单拆分镜、写手照写——漂移的直接凶手。
3. **AI 起草能力（draft.py 三件套）藏在主流程外**。只在用户主动打开空设定文件时提示；起步旅程卡四步（接模型→seed→织第一章→learn）里没有铺设定这一步。

## 决策记录（用户拍板，2026-07-06）

| 决策点 | 结论 |
|---|---|
| 方案取向 | A. 立项即铺底（复用现有 draft 能力编排进主流程，不做多步向导） |
| 立项卡红线 | 保持「loom 从不自动写它」；只做前置输入（建书框的平台选择写成 `平台:` 行——内容是作者填的，loom 只代为落盘） |
| 「大纲」概念 | 卡章纲即大纲，不新增总纲文件；长线写进世界观「冰山真相」+ 卡章纲前 5 章 |
| 起草时机 | 建书时填了「一句话设定」且模型就绪 → 进书房后自动起草（toast 提示进度）；没填设定或没 key → 等旅程卡点击 |
| 正文行内改 UX | 悬停行级 + 选区两条路，弹层就地内联，中央弹层退役 |
| 世界观/人物 AI 改写续写 | 做成编辑器动作（候选→确认落盘）+ 铺底引导讲清三种维护方式；不加维护模式设置项 |

## 设计

### 1. 引擎侧根因修复

- **书名注入**：`_build_user_prompt` 首行改「你要写的是《{书名}》第 N 章」，五棒全部生效（含重生成细纲路径）。`title` 折进续跑签名 `_cfg_bits`——改书名触发从设定师全量重跑，绝不拿旧书名的半截稿续跑。
- **占位内容不进 prompt**：knowledge 组装（`_read_file_items` 链路）对读进来的文本先剥掉占位提示行（「（占位示例:…）」式括注），剥完若无实质内容则整份跳过。不能按「文件含占位标记」整份误杀——立项卡是真内容（`平台:` 行）与占位提示混排的，半填状态必须让真内容照常进 prompt。draft 写侧的 `_is_blank_or_template`（防覆盖作者内容）语义不同，保持独立、两者共用占位标记常量。占位提示是写给作者的填写说明，不是书的设定。
- **卡章纲模板去毒**：`templates/外置大脑/卡章纲.md` 预填的三行通用示例行改为空行（`- 第1章:`），格式示范挪进顶部注释。没纲时大纲师按书名+题材+设定锚点自由发挥，好过照错误的纲执行。

### 2. 流程侧：立项即铺底

- **建书对话框**加两个可选字段（仍一屏，不做向导）：
  - 「一句话设定」文本框 → 存 `loom.toml` 新增 `idea` 字段（Config.idea），供起草用；
  - 「平台」下拉（起点/番茄/晋江/七猫/不选）→ 写入立项卡 `平台:` 可解析行（违禁词自检基线顺带就位）。
- **自动起草**：**仅建书成功后首次进书房**触发一次判断——idea 非空且后端就绪 → 前端自动调 `/api/brain/draft`（沿用现有 toast 反馈）；否则不动。之后打开这本书永不自动起草（起草本身只覆盖空白，但空跑也是一次计费调用）。
- **旅程卡四步变五步**：接入模型 → **AI 铺设定底稿（新）** → 喂样本 seed → 织第一章 → learn。新步骤点击调 `/api/brain/draft`；server 端 idea 为空时回退读 `Config.idea`；完成判定 = 世界观/人物/卡章纲三者中任一有实质内容（state 增加 `brain_ready` 字段，织章拦截同吃这个字段）。
- **织章拦截（兜底）**：点织章时若世界观/人物/卡章纲**全部**仍是占位 → 弹一次确认（主按钮「先让 AI 铺底稿」/ 次按钮「就这样写」）。填过任意一份即不拦——作者做过选择就不唠叨。

### 3. 起草质量顺手改良

`draft._DRAFT_SYSTEM` 现硬性要求「力量体系 7~9 级 + 金手指」，历史/都市题材会起草出不伦不类的设定。改为按题材取舍：玄幻修仙类保持现状；历史/现实类换成「时代格局 / 主角优势（信息差、先知、权柄）」。

### 4. 书房内容的 AI 协作（随时可改）

**正文行内改（局部重写的 UX 升级，引擎不动）**
- 触发一（新）：鼠标悬停正文某一行 → 行尾浮出轻量「改」入口 → 点开在**那一行下方**展开内联指令框，默认作用于这一行。
- 触发二（升级）：选中跨行文字仍走现有朱批浮条 → 但弹层从中央 overlay 改为**选区下方内联**。中央弹层退役。
- 对话框交互不变：指令框（可空）→ 重写 → 候选 → 用这个 / 再来一个 / 取消。
- 引擎侧复用 `rewrite_span` / `apply_rewrite`，**外科式快照同步铁律不变**（AI 改动必须同步 `.原稿`，learn 只学手改）。
- 实现风险记档：textarea 软换行下「视觉行 → 逻辑行」定位需镜像 div 度量；行悬停入口不得干扰正常打字（仅鼠标移动时显现、输入时隐藏）。

**世界观/人物 AI 改写、续写（新能力）**
- 打开世界观/人物文件时，编辑器动作区加两个动作：
  - **改写选中**：选中一段设定 + 指令 → 候选 → 确认后替换落盘；
  - **AI 续写**：按指令（可空）从文末续写一节 → 候选 → 确认后追加落盘。
- **口径**：不走写作指纹嗓音（设定是 what 不是 voice，ADR 0002 分工）。新 prompt 用设定师口径：按作者指令改写/续写，与该文件其余内容及书名自洽，不发明冲突设定。上下文 = 该文件全文 + 书名。
- **红线关系**：draft「只覆盖空白/占位」约束的是**自动**动作；改写/续写是作者**明示指令 + 逐次确认**，可以动真内容——所有权仍在作者（不确认不落盘）。不碰写作指纹、不涉 `.原稿` 快照（设定文件无快照概念）、learn 红线无涉。
- **立项时讲清**：铺底完成的引导文案点明设定文件的三种维护方式——随时手改 / 选中让 AI 改写或续写 / learn 后自动追加 [AI补充]。不加维护模式设置项。
- server 侧新增设定改写/续写端点（只生成不落盘，落盘走编辑器现有保存链路）。

### 5. 明确不做

- **违禁词**：模板已带默认清单 + 平台行调松紧，不动。
- **立项卡 AI 起草**：不做（红线保留，模板注释不改）。
- **总纲文件 / 多步立项向导**：不做，守「交互从简」。

## 书房内容生成时序（定稿口径）

1. **建书瞬间**（离线零调用）：立项卡（含平台行）/违禁词/文风参考/卡章纲空模板/中性写作指纹/选中题材 skill；世界观、人物目录为空；idea 存 loom.toml。
2. **AI 铺设定底稿**（一次 LLM 调用）：世界观（按 H2 拆目录多文件）+ 人物卡（一人一文件）+ 卡章纲前 5 章。初稿只覆盖空白，作者随后改。入口：建书自动（填了 idea）/ 旅程卡 / 空文件提示 / 织章拦截。
3. **每章写作中**：设定锚点、分镜细纲落 `正文/.细纲/`，审稿留痕落 `.审稿留痕/`（现有机制不变）。
4. **每章 learn 后**：卡章纲「AI回顾」、世界观/人物卡「AI补充·第N章」、写作指纹演进（现有机制不变）。

## 影响面与测试

分两个实现计划落地（一期不依赖二期，先止血漂移）：

**一期 · 立项即铺底（§1–3）**
- 改动文件（预计）：`agents.py`（书名注入/签名/占位过滤）、`parse.py` 或 `fsutil.py`（占位判定上提）、`draft.py`（题材自适应、复用判定）、`config.py`（idea 字段）、`scaffold.py`（平台行、卡章纲模板）、`server.py`（create 收 idea/platform、draft 回退 idea、state.brain_ready）、`webui/`（建书框字段、旅程卡五步、织章拦截、自动起草）。
- 测试：书名注入、占位过滤、brain_ready 判定、create 落盘（idea/平台行）各加单测；`tests/golden` prompt 快照重新生成（注意 Windows 路径归一化，3.1.0 踩过）。
- 兼容：老书 `loom.toml` 无 idea 字段 → 默认空串；占位过滤对已填真内容的书零影响（真内容不含占位标记）。

**二期 · 书房 AI 协作（§4）**
- 改动文件（预计）：新增设定改写/续写模块（设定师口径 prompt）、`server.py`（brain 改写/续写端点）、`webui/`（行悬停入口 + 内联对话框替换中央弹层、设定文件动作区、镜像 div 行定位）。
- 测试：设定改写候选不落盘/确认才落盘、续写追加位置、行定位软换行用例；正文行内改沿用现有 rewrite 快照同步测试。
