# 除虫闭环 —— 状态账本 · 连续性除虫 · 按建议改（设计定稿 · 2026-07-07）

**问题**：真书实测（用户修仙文,第 1-4 章）抓到五类跨章 bug 全部漏网：A 物品被消耗后复活（远古药胚第2章吞服、第4章又拿来炼丹）、B 人设跌份（金丹天骄对炼气主角无尊严下跪）、C 金手指规则数值漂移（因果锁定 100%→10%）、D 时间词粒度错（昨夜的事说成昨日）、E 越阶认知缺归因（一口叫破元婴禁制却没交代金手指推演）。用户现在靠把章节贴给第三方 AI 手工除虫。

## 根因（五路代码调查结论,三层结构性缺口）

1. **记忆无载体**：[AI回顾] 只记「摘要+伏笔」,[AI补充] 明文排除剧情流水——物品消耗/规则数值/故事时钟这类**状态**在外置大脑里无处可住。
2. **生成侧输入缺失**：写手 reads 不含人物卡/世界观,人设与规则只经设定师 ≤350 字有损转述（「一复述就漂」,ADR 0010 教训的残留面）;硬设定关键词表无「系统/因果」,系统流规则不受逐字保护。
3. **检测侧口径缺失**：质检硬伤清单无物品/时间条目;信息边界只查「不该知道」不查「该知道但没归因」;跨章输入只有上一章末 800 字——第 2 章对第 4 章的复审员完全不可见。

## 决策记录（用户拍板,2026-07-07）

| 决策点 | 结论 |
|---|---|
| 除虫触发 | 每章终稿后自动跑（设置可关）+ 编辑器「除虫这一章」按钮可对老章节重跑 |
| 分期 | 本期=A 状态账本+B 除虫关卡+C 生成侧口径+按建议改联动;字数五螺丝+拆章工具=四期 |
| 按建议改 | 进本期:报告条目一键定位涉事段落、预填修改示例进行内改写面板 |
| 账本写入 | 与除虫同一次 LLM 调用顺手产出（不依赖 learn、不多花调用）;人可改可删,AI 只追加新节 |
| 版本 | 0.3.5 |

## 设计

### 1. 状态账本（数据层,`loom/statebook.py`）

**文件**：`外置大脑/状态账本.md`（单文件,目录形态书同样单文件——跨章流水天然线性,同卡章纲理由）。scaffold 模板带 `>` 注释头（说明四类行、人可改可删、AI 只追加）,空模板经占位过滤不进任何 prompt。加入侧栏可选设定清单（OPTIONAL_BRAIN,文件存在即显示）。

**格式**：每章一节,四类行,每行带证据句:

```markdown
## 第2章
- [物品] 远古药胚:已吞服消耗(用于冲刷绝脉) | 证据:「张口将其吞入腹中」
- [状态] 江澈:炼气巅峰(自炼气七层突破) | 证据:「一举飙升到炼气巅峰」
- [规则] 因果锁定:感激度满100%触发,100%反馈修行成果 | 证据:「感激度圆满,因果锁定」
- [时钟] 章末:当夜深夜
```

**写入**：除虫扫描输出的第二段即本章状态行;write-once 按 `## 第N章` 节存在即跳过;AI 绝不改已有节（人写优先红线）。

**生命周期**：删章=该节摘除留底回收站;重编号=节标题章号键两段式搬运——全套参照 recap 的 strip/remap 现成模式,并入 chapters.py 删章/重编号的既有调用点。

**读出**：
- **「当前状态」块**直送大纲师/写手（`wants_hardfacts` 的同批工序）,放硬设定逐字块之后。确定性组装,不建实体库:近 8 章（沿 budget.WINDOW）四类行原样;远章只保留 `[物品]` 行中含「消耗/失去/损毁/赠出/用尽」关键词的行 + 全部 `[规则]` 行,`[状态][时钟]` 丢弃。块头写明「物品消耗/规则数值以此为准,不许复活已消耗物品、不许改规则数值」。空账本/空快照则整块不出现（golden 零扰动）。**不进续跑签名**（同 hardfacts 豁免:账本变更常态是随除虫追加,折进签名会每章冲 ledger;缺口与自愈路径同 agents.py 既有注释口径）。
- **脉络弹层第四页签「账本」**：studio.py 加解析器（按章列四类行）,前端 seg-btn 组加一枚。

### 2. 除虫关卡（检测层,`loom/continuity.py`）

**时机**：`run_pipeline` 终稿落盘后（违禁词扫描同排钩子）,吞异常绝不阻断出稿;`config.continuity_scan`（toml `[gate] "除虫"`,默认 true）可关。手动:编辑器动作区「除虫这一章」（章节文件可见）,`POST /api/chapter/debug` 锁内薄封装,对任何已写完章节可跑——老书从第 1 章逐章手动除虫即可补建账本。

**输入**：本章终稿 + 状态账本存活快照(1..N-1) + 近两章正文尾（各 ~1200 字）+ 卡章纲（折叠后,含[AI回顾]）+ 硬设定逐字块（含新增人物硬约束）。

**输出**（一次调用两段,`===除虫报告===` / `===状态入账===` 分隔）:
- 报告行格式:`- ⭐⭐⭐⭐⭐ | 物品 | 冲突一句话 | 本章证据:「…」 | 前情证据:第2章「…」 | 落点:正文 | 修改示例:…`。类别∈{物品,人设,规则,时间,衔接,其他};落点∈{正文,设定};无硬伤输出「通过」（沿 parse_verdict 的宽容判词协议扩展）。
- 入账行即 §1 四类行。

**双引擎**：LLM 复审走 `cheap_backend`（评估类既有路由）;另加两条**零 LLM 确定性检测**（continuity.py 纯函数,与 LLM 结果按〔类别+实体〕合流去重）:
1. 已消耗物品复用:账本 1..N-1 中带消耗类关键词的 `[物品]` 实体名（精确匹配,≥2 字）出现在本章正文 → ⭐⭐⭐⭐⭐ 候选。诚实局限:别名/简称靠 LLM 兜,精确匹配只保底。
2. 规则数值漂移:`[规则]` 行含百分比/倍数,本章正文同段落出现该实体名且数值不同 → 候选。

**落盘与呈现**：报告追加进 `.审稿留痕/第N章.md`（`## 除虫报告(非阻断,供你定夺)` 小节,行格式含双证据）;自动跑经新事件（events）在写完弹「除虫报告」卡片,手动跑由响应直接渲染弹层。**只报告,绝不自动改稿**（ADR-0006）。

### 3. 「按建议改」联动

报告弹层每条（落点=正文）带「按建议改」:打开该章 → 按「本章证据句」精确定位（唯一命中→选中所在段落;找不到→toast 请手动选中）→ 打开二期行内改写面板（chapter 模式,快照铁律不变）,指令框**预填修改示例**,你可改可跑。落点=设定的条目点击打开对应设定文件（LLM 在报告行给出文件名,给不出则打开状态账本）,不预填不自动改。

### 4. 生成侧口径（纯 prompt/关键词,根治 B/E）

- **人物硬约束进逐字块**:从人物卡（单文件 H2 节/目录一人一文件）确定性抽取标签含「底线/身段/命格/原则/禁忌」的行,拼进硬设定块新段 `### 人物硬约束(逐字遵守)`——与起草模板的「不可退让的底线」字段天然对齐;成长档案与 [AI补充] 块照旧排除。
- `_HARDFACT_KW` 补「系统」「因果」两词（`_SPOILER_KW` deny 仍压过）。
- `CRITIC_质检` 硬伤清单:新增 ⑥物品/状态连续性（对照状态账本与[AI回顾]）⑦时间连续性（时间词粒度与场景时刻一致）;⑤信息边界双向化:越境界/越阅历的认知展示,正文缺一句来源交代（金手指/情报/前世记忆）即硬伤。`评估自检.md` 同步补三条目。质检 gate 的 reads 补状态账本。
- `写手.md` 补一条:越阶认知必须带来源句。`大纲师.md` 细纲每场补「时间:」锚字段。
- `REVISE_质检`/`REVISE_去AI味` 补「篇幅保持原稿量级,绝不扩写」（字数螺丝中唯一顺手带入本期的一颗,堵回炉扩写逃逸口）。

## 影响面与代价

- 每章 +1 次便宜模型调用（除虫+入账合一,可关）。
- 大纲师/写手 prompt 新增状态块(仅账本非空时) + 硬设定新增人物硬约束段 → golden 视 fixture 内容可能重生成一次（白名单审计:新增块/新增段两类）;质检 gate reads/CRITIC 变化 → 老书首写全量重跑一次（签名,与前两期同类一次性成本）。
- 老书零迁移:无账本文件→除虫照跑（仅近两章视野）,逐章手动除虫可补建。
- **避让**:`budget.fold_supplements` 的标题级 bug 由另一会话独立修复中（task_bb07214d）,本期不触碰该函数;状态账本折叠是新逻辑不复用它。

## 明确不做（本期)

- 字数五螺丝其余四颗、拆章工具、AI 建议拆点（四期,spec 另立）。
- 时间全量时序推理（只做时间锚+粒度对照的窄口径）。
- 实体库/打分/向量/自动改稿（红线不变）。
- 除虫回炉（报告→按建议改的人工确认路径已闭环,不做自动回炉档位）。
