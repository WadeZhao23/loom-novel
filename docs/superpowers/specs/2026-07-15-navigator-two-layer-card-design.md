# 领航员两层卡 · 最终设计

> 综合四份设计 + 十六份评审。骨架取「槽位=书里已有的模板骨架行」,嫁接「slots 做独立字段」「格由 loom 定」「降级从状态降为属性」「端点分身份」四手,并修掉评审点名的全部致命洞(人物名字、一句话定位失明、大弧不存在、静默覆盖、跨版本游标、goto 短路、点击数翻倍)。

---

## 1. 一句话核心

**领航员不再自己挑「问哪一格」——loom 扫描外置大脑里已经存在的骨架行,确定性地算出「这段还空着哪些格」,把格定死,模型只负责为这一格提 2-4 条灵感候选;槽位清单作为独立字段随每一张卡下发,所以后端全死时用户看到的仍是一排可点的格,不是一个空白输入框。**

---

## 2. 槽位模型

### 2.1 核心断言(已对着模板逐行核实)

**槽位不用发明,它已经在盘上了。** 出厂模板通篇是同一个行形状:

| 文件 | 骨架行数 | 实例 |
|---|---|---|
| `世界观/金手指.md` | 8 | `- 代价·限制(至少一种硬代价,不能无敌到没冲突):` |
| `世界观/力量体系.md` | 3 | `- 等级(7-10 级,各级有可见的代价/异化):` |
| `世界观/地理与势力.md` | 3 | `- 安全岛(文明退守之地,内部有阶级矛盾):` |
| `世界观/冰山真相.md` | 1 | `- 这个世界藏着的终极秘密:` |
| `世界观/一句话定位.md` | **0** | 只有 `#` 和两行 `>` |
| `人物/主角·未命名.md` | 5 | `- 核心欲望(终极想要什么:复仇/变强/守护/登顶):` |
| `人物/反派·未命名.md` | 2 | — |
| `人物/配角·未命名.md` | 3 | — |
| `卡章纲.md` | 5 | `- 第1章:` … `- 第5章:` |
| `立项卡.md` | 平台行 + 4 个 H2 | `平台:起点` / `## 题材` |

而 `parse._EMPTY_ROW_RE`(parse.py:172)+ `_PAREN_SPAN_RE`(:176)**今天已经在读这些行**——`is_substantive` 靠它们判「空表单行不算实质」。它今天是个匿名谓词,我们把它升级成具名可寻址结构。

结果:槽位真相 = **书里的文件正文**(新书由模板铺出,老书/导入书按自己的样子读)。不是代码里的第四份表。作者删掉 `金手指.md` 的「反制方式」那行,那个槽就没了——槽跟文件走,不跟 loom 的清单走。这是 ADR 0011:19 反对的白名单的**反面**。

### 2.2 数据结构(新文件 `loom/slots.py`,约 150 行,零模型调用、零新依赖)

```python
@dataclass(frozen=True)
class Slot:
    id: str        # 稳定寻址 "<容器rel>#<键>"
    label: str     # chip 文案,≤10 字;实体容器带前缀「林潜 · 软肋」
    container: str # 容器文件 rel
    at: str        # "line" | "h2" | "row" | "filename" | "file"
    key: str       # 行/H2 的键;filename 槽 = "@name";file 槽 = "@body"
    hint: str      # 模板括注原文「至少一种硬代价,不能无敌到没冲突」→ 只喂 prompt,不上屏
    filled: bool   # 派生
    preview: str   # 已填值前 24 字(UI 灰 chip 上显示,让「回头改」看得见旧值)

def stage_slots(root: Path, spec: StageSpec) -> list[Slot]:  # 纯派生,零存储,每次现算
```

### 2.3 三种形状怎么统一:**容器 × 键**

| 简报说的形状 | 实际的统一表达 |
|---|---|
| **平格**(立项) | 容器 = `立项卡.md`;`平台` 是 `at="line"`(全库唯一可解析行),`分区/题材/对标意图/为什么选它` 是 `at="h2"` |
| **每实体一套**(人物) | **实体即容器**。同一套键在 N 个容器里各来一份,零嵌套。`外置大脑/人物/主角·林潜.md#软肋`,label「林潜 · 软肋」 |
| **列表**(卡章纲) | **键可枚举**。`卡章纲.md#第3章` 就是键带序号的普通 row 槽。不需要 list 特例 |
| **无骨架的文件** | `at="file"`,整份一个槽 |
| voice | 无容器 → 0 槽 → 静态卡(现状不变) |

三条必须写死的补充规则,每一条都对应一个评审挖出的致命洞:

**(a) `at="filename"` —— 实体的名字住在文件名里,不是行。**
`主角·未命名.md` 的 5 行里**没有「名字」行**,而模板第 3 行白纸黑字:「改文件名为「主角·他的名字」——文件名就是专名册里逐字直送写手的名字」。`_protagonist_done`(journey.py:102-113)硬判 `stem.startswith("主角") and 有分隔符 and "未命名" not in stem`。
所以:容器 stem 含「未命名」→ 产出**一个** `Slot(id="外置大脑/人物/主角·未命名.md#@name", at="filename", label="给主角起个名字")`,并**压住该容器的全部 row 槽**(没名字先别问软肋)。stem 已带真名 → 正常出 row 槽。
不加这条,作者点满主角 5 个 chip 全变绿,`_protagonist_done` 仍假,`writing_unlocked` 永久锁死,屏上没有任何一个 chip 能救他。这是 day-one 死路,不是打磨项。

**(b) `at="file"` —— 没有骨架行的文件不是不可见,是粒度粗一级。**
`一句话定位.md` 零骨架行,而它正是 `_premise`(journey.py:229-246)喂给**其余每一段** prompt 的那份设定。按纯 row 扫描它对槽位层永久失明,又因为同目录别的文件有槽位而触发不了任何兜底。
规则:容器里既无 row 也无可寻址 H2 → 整份一个 `at="file"` 槽,`filled = is_substantive(全文)`,hint 取文件里的 `>` 引导行。
这条同时是**导入书和散文化老书的自动降级路径**:作者把 `金手指.md` 重写成一段散文 → 该文件退化成 1 个 file 槽,不是 0 个槽。槽位层永不归零。

**(c) 不发明不存在的格。**
`大弧` 今天只存在于 `_land_card_lines` 的 digest 指令(journey.py:458)和测试夹具里,`卡章纲.md` 模板**没有这一行**;`开局钩子` 全库只出现在两句 goal 散文里。
裁决:① 模板补一行 `- 大弧:`(放在 `- 第5章:` 之后),新书即得一个真槽;老书没有这行 → 卡章纲段只有 5 个章槽,不合成假槽。② 开局钩子**不单列** —— `卡章纲.md:6` 原文已写「每行=这章完成什么+章末钩子」,它就是第 1 章的内容;`spec.goal` 与 `_STAGE_HINT` 的措辞同步改成「前 5 章一句话章纲(第 1 章带开局钩子)、全书大弧」。

### 2.4 排序与可达性

- **容器序**:`StageSpec` 新增 `slot_order: tuple[str, ...]`(文件 stem 优先序,如 `("一句话定位","力量体系","金手指","地理与势力","冰山真相")`)。未列的排后。**不用 `sorted(glob)`** —— 那是 Unicode 码点序,会把「冰山真相」排到「力量体系」前面,纯属抖动。
- **容器内序**:模板的声明顺序 = 作者视角优先级(金手指.md 是 类型 → 核心功能 → 触发条件 → 代价…),白捡一个排序真相。
- **跨容器轮转**:未填槽按 `slot_order` **round-robin** 交错,不是按文件深钻。否则金手指一个文件 8 行会霸屏前 6 位,「代价·限制」排第 12,新书上永远不可达。
- **可达性保证**:`slots` 字段带**全部**槽位(世界观 15 个,轻),前端只画前 6,余下折成「还有 9 格 ▾」本地展开。零新端点,零永久够不着的格。

### 2.5 代码侧新增的常量,以及**没有**新增的

新增只有 `StageSpec.slot_order`(排序,不是值域;文件不在表里只是排后面,不影响可达性)。

`_CARD_FIELDS`(journey.py:51)**保留但降级**:它今天有两个消费者,`_project_card_done`(:97)的第二句 `any(... for head, body in _all_h2(text) if head not in _CARD_FIELDS)` 是**救导入立项卡**的门禁兜底,不能动。所以它继续服务门禁谓词,**不再服务出题与落盘**——那两条改从 `立项卡.md` 的 H2 序读。诚实说法:这不是「六份收敛成一份」,是「四份决定落盘目标的表收敛成一份文件,剩下的表只管门禁」。

`_STAGE_HINT`(:55-61)**退回单一职责**:只做面板副标题。降级卡文案不再拼它——那正是「重复问已填的平台」的病根:chips 是文件派生的(平台预填「起点」→ filled → 不在未填清单),而句子是静态的。新题面用槽位现拼。

### 3.1 卡 JSON(`journey.next_card` → `/api/journey/card`)

2. **`tests/test_journey.py:119` 的 `options == []` 不破** —— 那条断言钉的是「降级卡没有模型候选」,它仍然为真。

### 3.2 关键反转:`slot=""` 时 **loom 自己挑第一个未填槽**,出内容卡

不是「先出一张槽位卡让你点」。理由:两层卡会让健康路径(有 key、网通,绝大多数时候)的每条设定多一次点击,而那次点击是为一条大多数时候不会发生的失败路径买单——税由 100% 用户支付。`slots` 字段已经让「换一格」随时可点,不需要把它做成一道必经的门。

副作用是好的:`tests/test_journey.py:78/89/98` 和 `tests/test_journey_usecases.py:12-25` 都是 `next_card(project, fake)` 无 slot 调用 → 仍然调后端 1 次 → **全部不破**。

### 3.3 模型输出契约(`templates/agents/领航员.md` 改后正文,逐字)

```
你是「领航员」——写作伙伴访谈的出题人。你的唯一职责:作者要定某一格设定,
你给他几个具体的、可以直接选的候选。

规则(红线):
- 你只出题、只给候选,**绝不替作者做决定**;候选是灵感提案,作者点了才算数,
  没点的一个字都不会进他的书。
- 候选可以是你替这本书想出来的具体设定(人名/宗门名/代价/钩子)——越具体越好,
  它们只是候选,不落盘。
- **只问任务里指定的那一格**,别问别的格、别把几格揉成一题。哪一格由 loom 扫描
  资料文件定死,不用你判断。
- 候选要具体到可以直接抄进设定文件,彼此差异大(不要「A更好/B也行」式假选项),
  一条一行。
- **先看【这本书的一句话设定】和【已经定下的设定】,候选必须紧贴这本书的题材与
  已定内容**——历史朝堂的书别给修真候选。用作者的题材语境说话,别用抽象创作术语。

输出格式(严格遵守,不加任何客套或解释,不要复述以上规则):
问:<一行问题,只问这一格>
- <候选一,15-40 字>
- <候选二>
- <候选三;共 2-4 条,至少 2 条>
```

对照今天的三处删除:
- 第 9 行「绝不发明设定」**删除**(用户拍板),「绝不替作者做决定」保留并加粗。
- 第 15 行 `格:<仅立项阶段输出此行,值取 平台/分区/…之一>` **整行删除**。格由 loom 定 → 枚举漂移在结构上不可能发生 → `journey.py:356-357`「格名不认识就悄悄改成题材」那个把漂移盖住的补丁失去存在理由(保留为 legacy 读侧,见 §5.4)。
- 第 21-22 行 `【无题】` **整段删除**。今天 `parse.py:209` 是**整段裸子串**且排在「问:」判定之前——模型只要复述一句自己的格式规范就自我降级。「无题可出」改由 loom 的 `stage_slots` 确定性算出,轮不到模型报。

同步收口:`CONTEXT.md:25` 的领航员词条也写着「绝不发明设定」,必须一起改,否则 CONTEXT.md(领域语言真相源)当场与模板互相打脸。

### 3.4 user prompt 新增段(拼在 `_premise` 之后)

```
要定的这一格:金手指 · 代价·限制
这一格是干什么的:至少一种硬代价,不能无敌到没冲突
这一格现在是空的。
```

`hint` 逐字来自模板括注——出厂模板本来就携带 prompt 级的创作意图,今天被 `_PAREN_SPAN_RE` 剥掉扔了。零成本,大收益,且不上屏(不用脚手架黑话污染 UI)。

**这是「让选项真出得来」的主杠杆**:模型从「挑题 + 编格名 + 出候选 + 判断无题」四件事收成「给定这一格,提 3 条候选」一件事。不是靠放松正则去捞烂输出。

### 3.5 HTTP 契约

```
POST /api/journey/card    {root, slot?}     ← JourneyCardBody 新增(今天收 RootBody)
POST /api/journey/answer  {root, answer}    ← 一个字段都不改
POST /api/journey/goto    {root, stage, skip}  ← 不动
```

- `slot` 必须校验 ∈ 当前段 `stage_slots` 的 id 集合;不在则当没传(**绝不信前端传来的地址**)。文件被外部改动导致 slot 失效 → 退回自动挑格 + `why="slot_stale"` 留痕。
- 缓存命中条件从 `cached["sig"] == sig` 收紧成 `cached["sig"] == sig and cached.get("slot","") == slot_req`。否则点「换一格」在文件没动时会命中缓存、静默吞掉 slot 参数。`:78/:89` 仍绿(两次都 `slot=""`)。

## 4. 降级路径

「降级」从**状态**降为**属性**:用户永远不会看到一张烂卡,只会看到一张少了模型候选的卡。

### 4.1 后端完全不可用(断网/没 key/超时/模型吐废话)

540px 居中大卡:

```
┌────────────────────────────────────────────────┐
│ 「世界观」还空着 15 格。先定哪一格?             │
│                                                │
│ [一句话定位] [力量体系·体系名称] [力量体系·等级] │
│ [金手指·类型] [地理与势力·安全岛] [冰山真相]     │
│                              还有 9 格 ▾        │
│                                                │
│ ┌ 或者直接写你想定的……                       ┐ │
│ └                                            ┘ │
│ [就这么定]  [重试出题]  [先答别的]              │
│ ▸ 为什么没出上题?                              │
└────────────────────────────────────────────────┘
```

- **全部可点。** chip 来自磁盘扫描,`next_card(slot=None)` 这条路径上没有 backend 调用。
- **绝不重复问已填的格** —— 已填的格是灰 chip,排在未填之后,不占前 6 位。`平台:起点` 模板预填 → filled → 立项段永远不再被当作待答格。今天那个「降级卡复读 `_STAGE_HINT` 问你已经填过的平台」的病根被**结构性拔掉**,不是靠改文案。
- 点 chip = `postJourneyCard(slot)` → 这次 prompt 锁死了格,命中率高得多。**不烧预算**(journey.py:292 只有成卡才 `asked+=1`)、**不进缓存**(:262 降级卡不吃缓存,`test_journey.py:322` 的纪律原样生效)。
- `▸ 为什么没出上题?` 展开 = `why` 码 + 留痕文件路径。

作者点了「金手指·类型」,后端仍然死着:

```
┌────────────────────────────────────────────────┐
│ 金手指 · 类型                                   │
│ 系统/重生记忆/随身空间/血脉觉醒/无金手指…        │  ← helper,来自他自己文件里的括注
│                                                │
│ ┌ 写下你的决定……                             ┐ │
│ └                                            ┘ │
│ [就这么定] [让伙伴提候选] [换一格 ▾]            │
└────────────────────────────────────────────────┘
```

叶子上没有确定性候选——**这是诚实的地板,不是失败**。金手指是什么只有作者知道,这正是产品意图那句「只有某些没有办法就是选项的内容,才让用户输入文字」。而且作者不会误落到这一屏:他是自己点进来的,输入框是**有指向的**(题面 + 说明 + 落点全确定),不是今天那个「领航员这次没出上题,直接写你想定的」的空茫大坑。「换一格」一步回到全可点的地方。

### 4.2 防花钱回路

降级卡不烧预算是今天的行为,而我们把它从死胡同变成了 6 个诱人可点的 chip。护栏三条:
- `_navAutoAsked`(app.js:867-872)语义不变:chip 点击是**用户显式动作**,不走自动分支,既不会被它挡住也不会绕开它循环。
- 点 chip 走 `/card`,不走 `goto` —— 所以**不会踩 journey.py:191 的 `j["asked"][stage] = 0`**(回头改=重开预算)。这是评审对某份设计最狠的一刀:slot chip 若复用 goto,不但被 `:175` 的幂等早返短路(点了没反应),拆掉早返又会让每点一次 = 预算清零 + 一次免费调用,`_MAX_QUESTIONS=4` 形同虚设。
- 成卡才计费的口径不变;`slots` 只是数据,点它不产生答案。

## 5. 落盘

### 5.1 两种点击,两种字节

| 点的东西 | 走哪 | 变成字节吗 |
|---|---|---|
| **槽位 chip**(「金手指 · 代价」) | `POST /card {root, slot}` | **不落盘**。它不是答案,是出题参数——不改文件、不烧预算、不清卡 |
| **内容候选**(「每吞一次折寿三天…」) | 进托盘 → `POST /answer {root, answer}` | 落盘 |
| textarea 自由文本 | 同上 | 落盘 |

槽位 chip 走出题端点是**语义正确**,不是省事:硬塞进 `/answer` 会逼后端在落盘路径上写「if 这其实不是答案: 转身去出题」,把 ADR 0013:31「答案单点注入」这条协议弄脏。走 `/card` 则四条协议形状分毫未动。

### 5.2 `_land_slot(root, slot_id, answer)` 按 `at` 分派

| at | 落盘动作 | 人写优先 |
|---|---|---|
| `line` | 平台行整行替换(journey.py:352 现有裁量:答题即作者本人拍板换值) | 不变 |
| `h2` | `_replace_h2_body`(现有:占位格→替换,人写内容→格尾追加,缺格→文末补格) | 不变 |
| `row` | 空行 → 填冒号后。**已填行 → 整行替换** | 见下 |
| `filename` | 重命名 `主角·未命名.md` → `主角·林潜.md` | 见下 |
| `file` | 文件无实质 → 追加在标题/引导行之后;有实质 → 追加到文末 | 只追加 |
| `""`(老卡/legacy) | 原样走今天的 `_land_field`/`_land_sections`/`_land_card_lines` | 不变 |

**`at="row"` 已填行整行替换的裁量**:未填的 row 才会出现在「待答格」里,所以这条只在作者**主动点了一个灰 chip 回头改**时触发——他看得见 preview 里的旧值,这是他本人拍板换值,同平台行先例。
明确**否决**「行已有内容 → 下方插缩进子行,照抄 `- [AI回顾]` 先例」:那个先例的语义是**物理隔离的 AI 子块**(CONTEXT:136),而访谈答案是**人写主体**(CONTEXT:25)。给作者拍板的答案套上 AI 内容的排版身份,读者按库内约定会误判来源,恰好违反它想守的那条「跟手写没区别」。

**`at="filename"` 的边界**(全系统后果最重的落盘——文件名逐字直送写手):
- 消毒复用 `draft._FN_BAD`;取答案第一行、截断 12 字。
- 目标已存在且无实质(占位) → unlink 后重命名。
- 目标已存在且有实质 → **不落盘**,抛 ValueError「已经有一个叫『林潜』的人物了」。前端 textarea 保留原文,答案不丢在盘上但也没进盘——这是全设计**唯一**一处拒绝落盘,写进 §12。

**`_land_slot` 与 `digest` 的关系**:`at` 路径上**零 digest 调用**——答案直接填进骨架行,产物就是模板行填上了内容,和作者自己填这一行**逐字一致**。这比今天让 digest 现造一个 `## 标题` 更贴 CONTEXT.md:136。裁量点:`len(answer) > 60`(作者写了一段散文)→ 退回今天的 digest 路径,避免烂排版。

### 5.3 一个必须先修的既有 bug:`_is_blank_or_template` 与 `is_substantive` 打架

`draft.py:79` 的 `_is_blank_or_template` 是**裸子串匹配** `PLACEHOLDER_MARKS`;`金手指.md:2` 是 `> (占位示例,换成你自己的。)`,而这行是 `>` 引导的,`_HINT_LINE_RE` 匹配不上、`is_substantive` 又会跳过它。

后果:作者用 chip 往 `- 代价·限制:` 里填了三格 → `is_substantive=True`(槽位层认为已填)→ 但 `_is_blank_or_template` **仍为 True** → 一点「AI 铺设定底稿」,`draft.py:146` 判可覆盖 → `atomic_write_text` 整份重写 → **作者拍板过的三格被静默抹掉**。

这**今天就是活的潜伏 bug**(`_apply_card_lines` 已经在原地填空章行,`卡章纲.md:7` 同样含「占位示例」)。我们把它从 1 个文件放大到 9 个文件,所以必须在**第一期**就修:

> `_is_blank_or_template(path)` 统一成 `不存在 or 空 or not is_substantive(text)`。一个谓词,两侧永不漂移。

这一刀有安全网:`tests/test_placeholder.py:19-42` 已经钉住了 `- 第1章:`/`- 体系名称:`/带括注的 `- 等级(7-10 级…):` 不算实质、`- 体系名称:玄元九阶` 算实质、`- 提刀护驾。`(无冒号真内容行)不被误伤。

副产品:`drop_unnamed`(draft.py:150-153)随之收紧——填过内容的 `主角·未命名.md` 不会被删。而有了 `at="filename"` 槽,它会被**重命名**,所以不会留下孤儿文件。两条修复互为前提。

**明确否决:落盘时剥掉占位括注。** 有人会想「填了就把 `(至少一种硬代价…)` 剥掉,读起来更像手写」。否决:`_PAREN_SPAN_RE` 是为**判空**写的读侧启发式,拿它当**改写用户文件**的授权,和 `journey.py:356`「格名不认识就悄悄改成题材」是同一种病——用一个近似判断去悄悄改用户的字。而且 `_is_blank_or_template` 修好之后,占位标记不再有任何写侧权力,没必要动。**一个字都不剥。**

### 5.4 「答案绝不丢」怎么保住

- `_land_slot` 是**新增的旁路**,不是替换。卡不带 `slot`(legacy `_prime_card` 造的卡、游标里的老卡)→ 原样走今天的 `_land_field`/`_land_sections`/`_land_card_lines`,**一行没动**。`tests/test_journey.py:171-283` 那 12 条钉死三种落盘模式的测试因此继续跑**真路径**,不是被改断言蒙混。
- `journey.py:356-357`「怪格名 → 落题材」**保留**,标 legacy(同 `EDIT_NOTE_SENTINEL`「读侧永不下线」先例)。新卡不带 `field`,所以它不再是活路径;但删掉它会让老游标里的 `field="力量体系"` 这种卡落进 `_replace_h2_body(text, "力量体系", …)` → 立项卡文末长出一个野生 H2。不删,零成本。
- `_apply_card_lines`(journey.py:446)的非章行处理要**补一条**:先试填空行(`^-\s*<键>[:：]\s*$` → 填),填不到再走今天的整行精确判重追加。否则模板新增的 `- 大弧:` 空行会永远填不上、答案重复堆在文末、该槽永远 `filled=False` 被反复重问。`test_journey.py:263-268`(`count(...) >= 2`)在新逻辑下仍绿——那个场景里已有一条**有内容**的大弧行,新行填进模板空行,计数仍 ≥2。

## 6. 前端

### 6.1 一个渲染器,两种形态,靠 wrap 吃两种宽度

`paintJourney`(app.js:806-899)不分叉。`centerMode` 那个开关今天只管画不画 chrome,卡体本来就是共享的——不为槽位层再撬开它。

```css
.jc-slots  { display:flex; flex-wrap:wrap; gap:6px; margin:8px 0 }
.jc-slot   { /* 小 chip,虚线边,淡色,≤10 字;label 由服务端保证短,不靠 CSS 截断 */ }
.jc-slot.filled { /* 灰 outline + ✓;点=回头改,title 显示 preview */ }
.jc-opt.cand { /* 今天的整行按钮 + 虚线边 + 「候选」角标 */ }
.jc-tray   { /* 已选托盘,选中项带 ①② 序号 */ }
.jc-helper { /* 题面下的灰副标题 */ }

```

- **两种 chip 的文本长度差一个数量级**(「深渊」vs「每吞一次折寿三天,寿元见底就再也吞不动」)——这是排版必须分叉的**物理原因**,不是审美偏好。槽位 chip 用 `inline-flex + wrap`;内容候选是 15-40 字的句子,只能整行。
- **形态不必分叉**:540px 一行排完 6 个 chip;320px 自然折成 2-3 行。无媒体查询。
- 「还有 N 格 ▾」本地展开,不发请求。

### 6.2 「灵感候选」怎么看得出来

红线改写(删「绝不发明设定」)之后,唯一挡住「AI 的字混进作者的书」的东西必须是**结构**,不是自律。三重:

1. **类型上钉死**:wire 上只有 `answer` 端点能落盘,而 `slots` 数组里的东西物理上到不了 `land_answer`。槽位 chip 点错了也只是换个格。
2. **视觉**:候选一律虚线边 + 「候选」小角标 + 卡头一行小字「以下是领航员替你想的候选,你点了才写进书」。
3. **动作**:候选 **不再即点即交**。点候选 = 进托盘,再点「就这么定」才提交。即点即交恰恰模糊了「这是候选不是决定」;托盘让「作者拍板」这个动作**看得见**。

这是本设计唯一一处主动增加点击的地方,理由是红线要求「必须看得出是灵感候选」,我认这个代价并主动要它。

### 6.3 零 JS 测试的风险,以及最低限度的护栏

仓库没有任何 JS 测试,而 `slots` chip 走 `/card`、`options` 走托盘这条**承重分派**零覆盖。不为此引入 JS 测试栈(超出范围、违「一人维护」)。护栏两条,写进实现清单:
- 手动 UI 冒烟必须**先钉 `loom_root`**(记忆里那条真书误写教训就是这么来的,而这次的误写面更宽)。
- 分派错一边的最坏后果是把「金手指 · 代价」当答案写进作者的文件——所以服务端**必须**校验:`answer` 端点收到的文本若逐字等于当前卡某个 slot 的 label,不特殊处理(那会是启发式),但留痕记一条 `why="label_as_answer"` 供事后发现。

## 7. 可靠性与可观测性

### 7.1 解析容错(`parse.py`,只放宽不收紧)

```
_CARD_Q_RE: ^\s*(?:\d+[.、]\s*)?[*_#\s]*(?:问题?|Q)[*_\s]*[:：]\s*(\S.*?)[*_\s]*$
选项行:     ^\s*(?:[-*•]|\d+[.、])\s+
```
吃下今天**全部降级**的 `**问**:` / `问题:` / `Q:` / `1. 问:` / `* ` / `• `。这大概率是降级率的第一大来源。

`【无题】` 哨兵(parse.py:209)**整个删除**,连同 `_CARD_F_RE`/`field` 的**写侧**产出。`_CARD_F_RE` 作为读器保留(老书的 `格:题材` 行被解析出来但 journey 不再消费),这样 `test_card_with_field_line` 的语义只是变成 legacy,不是消失。

`[:4]`(parse.py:217)**一字不动**。

**parser 保持纯读器**:有 `问:` 无 bullet → 如实返回 `{"question": …, "options": []}`,它没撒谎。

### 7.2 零选项卡:是 bug,治在 `journey`,不治在 parser

今天这是**三个 bug 叠在一起**:
- `app.js:896` 只认 `degraded` → 没有重试按钮
- `journey.py:292` 只要 `parsed` 为真就 `asked+=1` → 白扣一格 `_MAX_QUESTIONS`
- `journey.py:262` 只排除 `degraded` → 进 sig 缓存 → 文件不变就**永久钉死**这张烂卡

处方:`journey` 侧 `len(parsed["options"]) < 2` → 不成内容卡 → 降级卡(`options: []` + `slots` 非空)+ `why="few_options"`,不烧预算、不进缓存、`degraded=True` 于是重试按钮自动出现。一刀同时修好三重。

**为什么不治在 parser(让它返回 None)**:两个理由。① `parse.py` 模块头写死「读侧解析收拢在此」,「几条算够」是产品决策(2 条),该住 journey 的常量旁边。② 更决定性的:parser 返回 None 就**分不出 `unparsed` 和 `few_options`**,而这两者的处方完全不同(前者调正则/prompt,后者调 prompt/换模型)——今天全糊成一个 `degraded`,这正是用户「查不出原因」的根子。

### 7.3 无题:从模型嘴里挪进确定性计算

`stage_slots` 的未填槽为空 **且** `stage_done(root, spec)` → 该段真做完 → 标 skip、前进。模型没有报无题的接口,所以 `parse.py:209` 那个「复述规则即自我降级」的误判归零。

同时补一条确定性出口,替掉今天靠模型报无题才能走的那条路:**`focus` 段在 `asked >= _MAX_QUESTIONS` 且 `stage_done` 时自动释放**。否则回头改(`goto` 世界观,focus 钉住)的作者会被永久钉在一个 15 格永远填不满的段里——世界观模板 15 格,作者不可能填满,`未填槽为空` 这条出口对世界观/人物在真实书上结构性不可达。两条出口缺一不可。

### 7.4 DemoBackend

`backends.py` 的 `DemoBackend._pick` 加一条(与「文风分析器」「设定师」同列,排在通用兜底之前):

```python
if "领航员" in system: return _DEMO["nav"]
_DEMO["nav"] = "问:(demo 占位问题)\n- (demo 候选一)\n- (demo 候选二)\n- (demo 候选三)"
```

今天 `LOOM_DEMO=1` 的起书访谈**必然降级**(落到最后一行「(demo 占位)」→ 解析失败)。五行代码,让 demo 端到端全可点,顺带给测试多一条真实路径。与两层卡完全解耦,可以今天就单独提。

### 7.5 留痕:`.审稿留痕/领航员留痕.md`

`paths.py` 加 `NAV_TRACE_REL = f"{REVIEW_DIR}/领航员留痕.md"`。`REVIEW_DIR`(paths.py:21)在**项目根、不在外置大脑下**,定义原文就是「编辑留痕 / 关卡残留 / 各类提醒(人可读可改)」——领航员出题失败正是「各类提醒」。复用既有概念,比新造日志设施诚实。

只在**降级时**追加一块人可读 md,保留最近 20 条:

```markdown
## 2026-07-15 14:32:07 · 世界观 · 外置大脑/世界观/金手指.md#代价·限制
- sig: 9f2ac7e1b004
- 结果: few_options
- 后端: deepseek / deepseek-v4-flash
- code: —
- 原始回复(截断 500 字):
  > 好的!我来帮你分析一下这本书的金手指设计……
```

`why` 枚举:`backend_error | unparsed | few_options | slot_stale`。

三条硬边界:
- **绝不进 `.loom_state.json`** —— ADR 0013:30 要求游标薄且可丢弃,把 500 字原始回复塞进去会让「坏了当无、谓词重推」变成谎话。
- **绝不进外置大脑** —— 那儿的文件会被 agent 读到,会污染设定。
- **绝不派生任何状态**。谁拿它做「问过的题不再问」,就破了 `journey.py:1-7` 的「题从当前文件状态推导,绝不从问答历史推导」。这条写进 ADR 0015 红线。

成功卡不打点(体积 + 隐私:raw 里有这本书的设定)。`doctor` 顺带读它报「本书最近降级率」——把「查不出原因」变成一个可统计的数。

诚实标注局限:它只留得住最近 20 次失败,查不了完整历史。用户的诉求是「这次降级了查不出原因」,这一刀正好够。

## 8. 每个岔路的裁决

### 8.1 复合答案(「重生+复仇+宗门流」,7:3 主辅)

**要多选,但零声明、零白名单、零 API 改动。**

所有内容候选一律 toggle 进托盘,选 1 个是单选,选 2 个是复合。托盘里的**顺序即主辅**(第一个是主),UI 标 ①②。提交前**客户端 join** 成 `"重生 + 复仇 + 宗门流"`,`{root, answer}` 一个字段都不加。

- **不加 `picks: list[str]` 字段**:服务端拿到它做的就是 `" + ".join(picks)`,客户端在 POST 之前自己就能 join。为零新增服务端能力却要动 `server.py:355` + `usecases.py:331` + 一个测试。而且改成 `answer: str = ""` 可选会**删掉唯一能拦住陈旧前端的护栏**(缓存住的老 app.js 发 `{root, answer: "<chip 文案>"}` 会 422 而不是把槽位标签写进金手指文件)。托盘是 **UI 能力**,不是数据知识。
- **不给 slot 加 `pick: bool` 字段、不建 `_MULTI_SLOTS` 表**:「哪些格可多选」会长成一张会过时的手工表,跟 ADR 0011:19 否掉的白名单同源。
- **不设「最多两个」上限**:核过原文,`skills/题材/README.md:14` 的「最多两个」管的是**建书时往书里拷几份题材速查文件**(同文件第 3 行「init 时按你选的题材只会拷一份进书」、第 17 行「复合时把两份题材文件都拷进来」),约束的是 skills 目录,不是立项卡那行标签。而 `立项卡.md:13` 的示范答案白纸黑字就是**三个**标签。宁可不设规则,也不发明一条仓库自己都不遵守的规则。7:3 是 skills/题材 的写作方法论,不是槽位层该知道的知识。

### 8.2 零选项非降级卡

**是 bug,三重的,治在 journey。** 见 §7.2。

### 8.3 4 个选项上限

**同一个数字,两条不同的规矩,不共用常量。**
- 内容层的 `[:4]`(parse.py:217)留在原地:它是**信任边界**,防模型话痨。`test_options_capped_at_four` 不动。
- 槽位 chip 上限 **6**,住前端:它是**渲染约束**(320px 弹层塞不下),而且槽位是确定性的、逐字来自盘上文件,不需要防。

凑巧同为个位数不代表同源——写成一个常量会在某天改弹层宽度时误伤模型护栏。而且**槽位不能截断**:世界观 15 格、卡章纲 6 格,截到 4 就是「作者永远够不到第 5 格」= 功能残缺不是节制。所以 chip 是「画前 6 + 展开剩下的」,不是「取前 4 丢掉其余」。

### 8.4 选项身份

**分开,靠端点分,不靠字段分,零新端点。**
- 槽位 chip → `POST /api/journey/card {root, slot}`(出题参数:不落盘、不改文件、不烧预算)
- 内容候选 → `POST /api/journey/answer {root, answer}`(原封不动)

「加个 `type` 字段区分」是错的抽象:槽位 chip 根本不产生答案,硬塞进 `/answer` 会逼后端在落盘路径上写「if 这其实不是答案: 转身去出题」——那是把导航伪装成数据,并且把 ADR 0013:31 的「答案单点注入」变成「answer 端点有时候不落盘」。

**明确不走 `goto`**:`goto` 的语义是「聚焦 + 重开本段预算 + 作废待答卡」三件事捆在一起,而且 `journey.py:175-176` 对当前段幂等早返(chip 点了没反应)。`/card` 本来就是出题端点,今天无参只是因为它还不需要参数。

### 8.5 两种形态

**渲染器不分叉,靠 flex-wrap 吃两种宽度。** 见 §6.1。视觉分层直接就是语义分层:小虚线 chip 读作「去那儿」,整行按钮 + 候选角标读作「就它了」——用户不用被教,尺寸差异自己会说话。

### 8.6 模板同步

**不同步、不检测、不加版本号,靠设计让老书无痛。**

新契约是旧契约的**真子集**:老书的 `agents/领航员.md`(scaffold.py copytree 只在 init 时拷一次)还会吐 `格:` 和 `【无题】`。新解析器里 `格:` 行被读出但 journey 不消费(落盘目标是 loom 选的 slot);`【无题】` 那行既不是 `问:` 也不以 `- ` 开头 → 被忽略;若模型只输出这一行 → `unparsed` → 降级卡(有 6 个真 chip + 重试 + 换一格),不是死锁、不是崩溃、不是循环。

老书拿到什么:**槽位层 100% 照常**(它根本不过模型)、**定址落盘 100% 照常**(格是 loom 选的)、只有内容层的候选质量差一点(prompt 少一句「只问这一格」)。

**这就是「格由 loom 定」这个反转最值钱的地方**:它把「模板同步」从一个要新造的机制变成一个不存在的问题,而且对所有未来的 prompt 改动都成立。反过来说,如果把格的选择留给模型,老书就**必须**同步,那才需要新造一个升级机制。

`doctor` 加一条**软提示**(不自动覆盖——那会毁掉作者手改过的 agent,违「人写优先」):「你的 `agents/领航员.md` 是旧版,新版会给更贴题的候选,可手动覆盖」。

同类的:`卡章纲.md` 新增的 `- 大弧:` 行只对新书生效,老书没有该槽。写进 §12。

## 12. 诚实的取舍

**1. 槽位层的确定性是借来的,债主是作者本人。**
地基是出厂模板的骨架行。但 `金手指.md:2` 原话就是「(占位示例,换成你自己的。)」,`skills/题材/README.md:4` 是「这是参考,不是铁律——直接手改成你这本书的样子」。**Loom 反复地、正确地、大声地叫作者把这些骨架砸了重写。** 作者照做那一刻,那个文件退化成 1 个 `at="file"` 粗槽——不再是 0 个(这是 `at="file"` 规则买到的),但粒度粗了一整级:他失去的是「代价·限制」这种精确的格,拿回的是「金手指(整份)」。
所以:**这个设计对刚 init 完的空书最强,对被真正用起来的书弱一级。** 导入的书(ADR 0011 通道明说原样落盘、从来没有骨架行)从第一天就只有 file 槽。而那恰恰是投入最深的作者。这个退化是渐进的、不是断崖,但它是真的,而且**是静默的**——作者只会觉得「以前小格子多,现在少了」。

**2. 我用一个静态序换掉了一个动态判断。**
「格由 loom 定」是设计的支点,但它牺牲了模型最擅长的那件事——判断这本书**此刻最该问什么**。模板声明顺序是模板作者拍脑袋定的通用序,对崇祯的历史书未必对;今天的模型至少能看着全书资料挑「下一个最值得问的决策」(`领航员.md:10` 原话)。轮转 + 全清单展开缓解了「够不着」,但缓解不了「顺序不对」。这笔交易在起书访谈(作者对自己的书还没想清楚,任何合理顺序都行)大概率划算,在回头改(作者心里有明确的洞)大概率吃亏——而回头改正是 `goto` 存在的理由。§11 的盲评实验就是为了在三期之前把这笔账算清。

**3. 「一份真相」是超卖的。**
我宣称把手抄副本收敛成文件本身,但没完全做到:`_CARD_FIELDS` 还活着(服务门禁谓词)、`_STAGE_HINT` 还活着(面板副标题)、`spec.goal` 还在喂 prompt。诚实的说法是「**决定落盘目标的表**从四份收敛成一份文件,剩下的表只管门禁和文案」——比今天好,但不是那句漂亮话。

**4. 三处放弃:**
- **spawn 槽(「+新配角」)砍掉。** 简报从没要过它,它要付第 6 个 `at` 变体 + 一个 `entity_types` 表(打脸「真相=书里的文件」)+ 一条没写手的落盘路。作者要第二个配角,自己建文件即可自动成容器。
- **`picks` API 字段砍掉。** 托盘作为 UI 有硬论证,API 字段没有;而且改 `answer` 为可选会删掉唯一能拦住陈旧前端的 422 护栏。
- **`- 大弧:` 只对新书生效。** 老书的卡章纲段只有 5 个章槽,大弧走今天的 digest 兜底路径。不为它造模板同步机制。

**5. 唯一一处拒绝落盘。**
`at="filename"` 槽遇到同名且有实质的人物文件 → ValueError,答案不进盘(仍在 textarea 里)。「答案绝不丢」在这一点上是**不丢在屏上,但没落在盘上**。理由:文件名逐字直送写手,把两个人合并成一个文件的后果比一次报错重得多。

**6. 前端零测试。**
`slots` 走 `/card`、`options` 走托盘这条**承重分派**,以及托盘 join、`jcBtn` 的 MouseEvent 陷阱,全部零自动化覆盖。不为此引入 JS 测试栈。手动冒烟**先钉 `loom_root`** —— 记忆里那条真书误写教训就是这么来的,而这次的误写面比那次更宽。

**7. 点击数的税。**
内容候选从「即点即交」变成「进托盘 → 就这么定」,每条设定多一次点击,由 100% 的用户支付。这是为红线改写付的对价:删掉「绝不发明设定」之后,「作者拍板才算数」这个内核不能再靠文案提醒,它得靠一个**看得见的动作**。如果实测下来作者觉得烦,该先砍的是托盘、不是槽位层——因为可以让托盘在只选 1 个候选时把「就这么定」高亮成默认动作(回车即提交),这条捷径不牺牲任何东西。留作 v2,不在 v1 里赌。
