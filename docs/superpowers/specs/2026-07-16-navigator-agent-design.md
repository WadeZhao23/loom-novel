# 领航员对话 agent(书房伙伴)设计

> 状态:**草案,待作者审**(已过一轮 4 镜头×对抗验证的多 agent 评审,33 条 findings 全部吸收)。
> 关系:取代〔[两层卡设计](2026-07-15-navigator-two-layer-card-design.md)〕的 **UI 部分**(chip/托盘作废),继承其两件**设计遗产**(均为蓝本待实现、非现成代码)——槽位扫描器(§5「看地基」,`loom/slots.py` 待新建)与三期定址落盘表(§6);可靠性一期(navigator-reliability-p1,449 绿)是本设计的地基:解析容错手艺、留痕/why、demo 分支全部复用。

## 0. 已拍板的决策(作者,2026-07-16)

1. 领航员升级为**对话式 agent**:对话循环 + 工具(手和脚)+ skills,悬浮球随时可召。
2. **agent 吞掉访谈**:起书也走对话,问题卡片机退役。
3. 工具范围全要,分期交付:读+改设定(拍板落盘)→ skills 方法论 → 驱动 loom 动作 → 对外调研(skills 并入 v1a,见 §11)。
4. **对话记录存书内**(`.伙伴对话/`),与书同备份、同迁移。
5. 上下文架构:分层组装 + 拉取优于塞入 + 注册表单一真相 + 上下文≠状态。

## 1. 一句话核心

**领航员从「出题机」变成「书房里的伙伴」:一个架在五后端共用单轮 `complete()` 之上的手搓对话循环,靠文本行式工具协议读书、看地基、提候选;作者拍板才落盘,落盘走定址通道;对话是可丢弃的上下文日志,书的文件仍是唯一真相。**

## 2. 产品形态

- **两个形态,一个对话**:居中大卡位(起书/无章节时)与右下悬浮球弹层(写作中途)渲染**同一条对话流**。悬浮球保留地基进度头(立项✓/世界观✓/…,由谓词现算)。
- **起书**:建书 → 打开 → 伙伴主动开场(读环境快照,直奔还空着的地基格发问)。原**四段访谈(每段封顶 4 题)**的地基问题改为对话中逐个问,答案经拍板落盘,门禁谓词(`writing_unlocked`/`missing`)照常解锁。
- **开场的触发与幂等**(吸取 `_navAutoAsked` 的教训):打开面板一律先 `GET /api/partner/history`;**仅当** history 为空(`当前.jsonl` 不存在或无事件)**且**书无章节(居中态谓词,复用 `journey_state`)时,前端自动调 `say {root, text: ""}` 触发开场;其余情况只渲染 history,绝不自动发轮。服务端对空 `text`:不追加 user 事件(不留永久空消息);若 `当前.jsonl` 已有事件则空 text 请求按 no-op 返回——双保险防重复开场。
- **中途**:任何时候点球即聊。**与织章的边界**:织章期间对话照聊(读文件/看地基/提候选全可用——say 不碰书写锁,§10),只有**拍板落盘**和**开工序**要等本次织章跑完(确认键置灰:「织章进行中,跑完即可拍板」)。先例:`PUT /api/file` 的锁豁免红线(server.py「整章生成几分钟,期间必须能存世界观」)——作者面向的操作必须在织章期间存活,本设计同一口径。
- **门禁引导卡**的「去补齐」按钮改为打开对话。

## 3. 架构总览:每轮重建,两轮之间零挂起

```
一轮 = 一次 HTTP 往返(POST /api/partner/say,ndjson 流):

  读 .伙伴对话/*.jsonl 尾部 + 读书文件(谓词/槽位现算)
      │
      ▼
  assemble() 纯函数拼 prompt ──→ backend.complete(system, user)
      │                              │(on_chunk 增量,经行缓冲纪律转发,见 §5.2)
      │            ┌─────────────────┘
      ▼            ▼
  解析回复:说话段(发给作者) + 至多一个「用:」工具块
      │
      ├─ 有工具块:执行 → 结果追加 jsonl + 回喂 → 再 complete(≤6 次/轮)
      └─ 无工具块:本轮终结,追加 jsonl,进程归零
```

**核心不变量**:两轮之间进程里没有任何活着的计算。关机三天回来,下一轮从 jsonl 尾部 + 文件现状重建一切。崩溃语义平庸化:任何时刻杀进程,最多丢正在写的一行;没有「恢复失败」,因为没有要恢复的执行栈。人在回路(拍板)发生在**两轮之间**,不是挂起的 interrupt——ADR 0013「答案单点注入」的形状平移。

**后端协议零改动**:循环每轮把对话记录重拼进 `user`,五后端的单轮 `complete(system, user, max_chars, on_chunk)` **签名与调用约定**原样使用。两处**内容级**改动必须诚实列出:

1. **CLI 后端的反 agent 护栏需要伙伴变体**:`ClaudeCodeBackend._GUARD`/`CodexBackend._GUARD`(backends.py)现在每次调用都注入「你是纯文本生成函数……禁止调用或提及任何工具、禁止反问」——伙伴通道下它会掐死文本协议工具块和「向作者发问」这个本职。改法:`complete` 加 keyword-only 参数(如 `agent_mode=False`)或伙伴专用护栏变体——**允许**按文本协议输出至多一个 `用:` 块、**允许**反问;**保留** `--allowed-tools ""` 与 codex read-only 沙箱(真实工具执行永远由 loom 服务端做,CLI 只产文本)。
2. **DemoBackend 罐头升级**(§8)。

## 4. 上下文工程

每轮 `assemble(对话尾部, 书文件) → (system, user)`,纯派生、零隐藏状态、FakeBackend 离线可测。

```
┌─ 稳定前缀(依赖的文件与代码不变时逐字节不变,吃 DeepSeek 前缀缓存)─┐
│ ① 人设与红线   agents/领航员.md(书内可改,包内兜底——现有机制)      │
│ ② 工具契约     代码侧注册表渲染(作者不可改,见 §5)                  │
│ ③ skills 索引  每轮现列(标题+一句话),新增 skill 立即可见;           │
│                正文靠「读文件」工具拉                                │
├─ 动态后缀(每轮从文件现算,绝不存)───────────────────────────────┤
│ ④ 环境快照     书名/一句话设定/门禁与各段完成度/未填槽位摘要         │
│                /章节数(≤400 字,构成见下)                            │
│ ⑤ 对话尾部     jsonl 尾部,预算内截断;更早的进 UI 不进 prompt        │
│ ⑥ 工具结果     单条封顶,超限提示 agent 用范围参数重取                │
└──────────────────────────────────────────────────────────────────┘
```

四条原则(渊源见 §14):

1. **拉取优于塞入**:不预载外置大脑正文,agent 用读工具按需取。一本书几十个 md,文件系统就是索引,**不建向量库**——理由与重议触发器见下。
2. **前缀稳定是「文件不变则字节不变」的弱保证,不是冻结**:①③ 恰是书文件(人设可改、skills 可增),作者中途改它们**立即生效**、代价是一次缓存失效——这与 `journey._sig`「文件一动签名即失配」是同一条纪律,也是产品语义上对的(改红线不该冻结到下一段对话)。DeepSeek 前缀缓存只要相邻请求前缀一致即命中;CLI 后端不受益也不受损。
3. **注册表单一真相**:工具在一处声明,同时渲染 prompt 契约段、驱动解析、驱动分发——prompt 与 parser 永不漂移(`STAGES` 表模式的延伸)。工具契约**刻意不进** `领航员.md`:两层卡设计核实过——模板只在 init 时 `copytree` 拷进书一次、此后永不同步(书内优先、包内兜底,journey.py:200-205),包内格式改动到不了老书;注册表运行时渲染让每本老书永远拿到当前语法。人设归作者,协议归代码。
4. **上下文≠状态**:完成度/游标/门禁永远从书文件推导。**把 `.伙伴对话/` 整个删掉,书完好无损**——未拍板的候选随之作废(它本来就还不是书的一部分),伙伴失忆,仅此而已。唯一被程序读回的对话事件是**待确认 proposal 行**(§6)——它是 ADR 0013 薄游标「待答卡缓存」的 jsonl 同构:可丢弃、不派生完成度。

**环境快照的构成**(与「看地基」工具的分界由共用数据源保证):快照与看地基共用同一个 `stage_slots()` 扫描器(单一真相,绝不写第二个扫描器);快照 = 其只读投影——每段一行「段名 未填N/总数:前K个未填的 容器#键」,`slot_order` 序,无 preview/hint;看地基 = 全量明细(含 filled/preview)。伙伴开场能「直奔空格发问」靠快照就够,细看某格才用工具。

预算常量(代码侧,不进 loom.toml):前缀 ≈2k 字 / 快照 ≤400 字 / 对话尾 ≤12k 字 / 单条工具结果 ≤3k 字 / 每轮工具调用 ≤6 次 / 单次 complete `max_chars` ≈1200。

**为什么不建向量库**(裁决留档,防日后反复重议):

- **两个前提都不成立**。向量检索解决「语料大到没法全看、又无结构可循」时的语义近似召回。Loom 的外置大脑是几十个百来 KB 的 md(没有大海),且文件名即语义索引(`主角·林潜.md`/`金手指.md`)+ 槽位扫描器给出容器×键的精确结构(标签是作者亲手贴的)。**向量检索是「无结构时对结构的近似模拟」,而这里结构真实存在**——用近似替精确是倒退。
- **它是唯一没法「坏了当无、现场重算」的东西**。嵌入索引是文件内容的派生缓存,作者改 `世界观.md` 即馊——要么每次保存重嵌(要嵌入模型/花钱/管失效),要么按旧语义检索。这与 langgraph checkpoint 同病:违「文件即真相、读时现算」宪法,是全系统里唯一没法用 sig 哈希+谓词重推兜底的第二真相。
- **依赖税同款**。五后端里 Claude/Codex CLI 无 embedding 端点、demo 无网;本地嵌入模型给 25MB 包塞几百 MB,外加 faiss/chroma 的 PyInstaller 税。
- **精确性这一课仓库已用真 bug 学过**([[ADR 0010]] 硬设定逐字旁路:力量体系被复述两道即漂出「一阶0级」)。写小说一致性要的是逐字正确,而切块+语义召回天然带漂移。
- **正文长大已有确定性解**。真正会长的只有正文(几百章),跨章记忆走卡章纲脊柱 + [AI回顾] 块 + 状态账本滚动窗(人可读、作者可改的文件,非黑盒索引,即 Manus「file system as ultimate context」)。模糊召回的第一解是给注册表加 `搜正文`(grep)工具——零基建、确定性、专名直接命中。
- **重议触发器**:当且仅当 (a) `搜正文` grep 工具在真机上明显不够(作者的模糊回忆 grep 命不中),**且** (b) 正文规模已过数百章——届时嵌入索引作为**可丢弃的派生缓存**(删了 grep 重建、绝不当真相、绝不进 sig/门禁)再议。在此之前,加向量库 = 违 ADR 0013 无第二真相红线。

## 5. 工具系统

### 5.1 注册表

```python
@dataclass(frozen=True)
class ToolSpec:
    name: str                 # 「读文件」
    params: tuple[str, ...]   # (「路径」,)
    desc: str                 # 渲染进 prompt 的一句话
    handler: Callable         # (root, **params) -> str(结果文本)
    mutates: bool             # True 者必须走确认流,v1a 仅「提设定」
```

### 5.2 文本协议(与解析器共置 parse.py,同 `问:`/`格:` 手艺)

模型每轮回复 = 可选的说话段 + 至多一个工具块:

```
先说给作者听的话(可空)。

用:读文件
路径:外置大脑/世界观/金手指.md
```

- `用:<工具名>` 触发;其后连续的 `键:值` 行为参数,空行或文末终止。
- 键值行宽容:全/半角冒号、markdown 装饰剥离(复用一期放宽后的正则手艺)。
- 每回复只执行**第一个**工具块;工具块之后的尾巴文字**丢弃**,并作为一条 meta 记录进对话 jsonl(不进降级留痕——不是失败,是协议裁剪)。
- 解析失败 → 把「格式错误+契约原文」作为结果回喂,计入轮内次数;连续 2 次失败 → 终结本轮并留痕(`why=tool_unparsed`)。
- 回复无工具块 = 本轮终结。

**流式纪律**(critical:协议行绝不许漏到作者屏幕上):服务端对 `on_chunk` 增量做**行缓冲**——整行落定后检查:仅当该行(剥装饰后)以 `用:` 开头才停止对外转发;在此之前的所有整行**照发** `assistant_text`(包括含冒号的普通说话行,如「书名:……」——中文对话里键值形状的行常见,不许按「疑似键值」预扣,否则吃掉合法说话)。自首个 `用:` 行起,后续增量全部进工具块缓冲,`complete` 返回后按上述规则解析;流结束时缓冲区残留的未终结行若非 `用:` 行,随最后一个 `assistant_text` 冲出。注意 chunk 可在行中截断(`用` 与 `:` 分两个 chunk 到达)——行缓冲天然免疫,但测试必须钉住(§12)。若实现中发现行缓冲复杂度不可接受,退路是「含工具轮次整段发、不做 token 级流」——但须同步删除 §15.5 的「流式让等待可见」缓解叙述。

### 5.3 v1a 工具清单(刻意只有三个)

| 工具 | 参数 | 作用 | 写盘 |
|---|---|---|---|
| 读文件 | 路径(白名单见下) | 返回正文;超 3k 字截断并提示带 `起行/止行` 重取 | 否 |
| 看地基 | (无) | 槽位扫描器全量输出:容器×键、filled/空、preview——实现二期蓝本 §2 的扫描器(设计遗产,`loom/slots.py` 待新建) | 否 |
| 提设定 | 落点(`容器#键`)、内容 | 产出**候选卡**(proposal 事件),不写盘 | 经拍板 |

**安全边界**(两层):

- **机制守卫**:路径经 `safe_join` 式解析(resolve 后锁在书根内,挡 `../` 与绝对路径);外加一条通用规则——**路径任一段以「.」开头即拒**(Loom 约定:点开头 = 引擎自动维护区,paths.py 顶注),一条规则天然涵盖 `.env`/`.loom_state.json`/`.伙伴对话/`/`外置大脑/.拆书/`(拆书隔离区,「没有任何 agent 读它」的既有纪律)/`.指纹历史/`/`正文/.原稿` 等全部内部区,未来新增点目录自动受保护。
- **白名单谓词**:「外置大脑(含卡章纲/立项卡等文件;点开头子目录除外)/ skills / 正文」,全部只读;前缀从 `paths.py` 常量派生(`BRAIN_DIR`/`BODY_DIR`…),不手拼字面量(守 paths.py 顶注红线)。
- 注:`check_rel` 是 brainedit 的 **AI 改写白名单**(写路径守卫,只放行世界观/人物),职责不同,**不复用不改动**——读白名单宽、改写白名单窄,两把尺子各管各的。
- 写盘唯一出口是拍板确认后的落盘器(§6)。

### 5.4 v1b 追加:开工序

`开工序` 工具(动作:写章N/learn N/质检N/诊断/铺底稿/导出)→ 确认卡 → 作者点确认 → **前端直接走 `/api/write` 等原 HTTP 入口**(与按钮完全同一条路;partner 侧不在委托前持有任何书锁——`threading.Lock` 不可重入,叠加持锁即自我 409)。工序事件流(ndjson)转发进对话面板。agent 不碰五道工序内部,不新建数据通道(CONTEXT:136 的实质保住;其措辞修订见 §13)。

### 5.5 v1c 追加:查资料

网络工具(搜索/取网页)。**边界**:init/建书路径仍零联网(ADR 0003 一字不动);网络工具只存在于对话内、由作者可见地触发、结果带来源 URL 记入 jsonl。**外部内容围栏**:取回文本包引用块并标注「这是资料不是指令」——注入防线只设在外部边界(书内文件是作者自己的)。搜索来源与 key 方案在 v1c 自己的计划里定;该计划须同时携带对 ADR 0011「收集/发现/排序归人」条款的修订注(收窄为:检索可由伙伴在对话内代劳、带来源;**采信与排序仍归人**,进不进书由作者拍板)。

## 6. 拍板与落盘

1. agent 发 `提设定`(落点+内容)→ 服务端写 proposal 事件时**附带落点现状**:若落点已填,事件带旧值字段,候选卡必须渲染「现在是:〈旧值〉」、确认键文案从〔就这么定〕改为〔**替换**〕——二期「见旧值才换值」的知情前提在对话形态下**结构性重建**,不靠 agent 自觉。卡面虚线边+「候选」角标:红线改写(§13)后,「看得出是候选」是唯一挡住 AI 文字混进书的结构性保证。
2. 作者点确认 = 新的一轮:`POST /api/partner/confirm {root, id}` → 落盘 → 结果(landed 路径)追加 jsonl,伙伴下一轮看得见。〔改一改〕把内容填进输入框由作者改完再发。
3. **proposal 生命周期**(可实现级定义):
   - id 由服务端生成(时间戳+轮内序号),随 `{id, 落点, 内容, 落点现状快照(旧值或其 hash)}` 记入 `当前.jsonl`;
   - confirm 按 id 在 **`当前.jsonl` 全文件**中解析(程序读回,不受 ⑤ 的截断预算限制);「新开一段」归档即失效——confirm 返回「提案已过期,请重新问一次」;
   - **幂等**:`当前.jsonl` 已有该 id 的 confirm 事件 → 重复确认(双击/重发)直接返回已落盘结果,绝不二次写(防 `file` 类「只追加」落两遍);
   - **改文件即分叉**:确认时落点现状与快照不符(作者拍板前手改了那一格)→ 拒绝落盘并告知,伙伴下一轮看见结果重新提——文件真相赢,永远。
4. **落盘器实现三期蓝本 §5 的定址表**(被本设计拉进 v1a):

| 落点类型 | 动作 | 既有规则沿用 |
|---|---|---|
| line(平台行) | 整行替换 | 「答题即拍板换值」裁量 |
| h2(立项四格) | `_replace_h2_body` | 占位→替换,人写→追加;**ADR 0014/0011 的修订轨迹见 §13,不许沉默着踩** |
| row(骨架行) | 空行填冒号后;已填→整行替换(卡面已渲染旧值,作者知情替换) | 括注一个字不剥 |
| filename(人物命名) | 重命名 `主角·未命名.md` | 目标已有实质 → **拒绝落盘**并告知(全设计唯一拒绝点) |
| file(无骨架文件) | 追加 | 只追加 |

5. **人写优先/答案绝不丢**:tests/test_journey.py「答案落盘」区段的 `test_land_*` 全套(现 13 条,不硬编码行号——它会腐烂)**断言语义**全部保住——落盘器是这些规则的新调用方,不是替代;测试迁移口径见 §12。

## 7. 对话存储 `.伙伴对话/`

- 位置:书根下,与 `.审稿留痕` 同级。**不进外置大脑**——五道工序永不读它,不污染设定。
- 一条对话一个 `当前.jsonl`;「新开一段」把它改名为 `归档-<时间戳>.jsonl`(一本书一条连续对话,伙伴是一个人不是 N 个聊天窗)。
- 事件行:`{"t": "user"|"assistant"|"tool"|"result"|"proposal"|"confirm"|"meta"|"summary", "ts": …, …}`。
- 写:每事件单行 append(单次 write);读:坏行跳过(ledger「坏 JSON 当无」同款)。归档/压缩时才整文件原子重写。
- `loom backup` 自动包含;删除整目录 = 伙伴失忆 + 未拍板候选作废,书无恙。
- 压缩(v1b):对话尾超预算时把更早轮次摘要成一条 `{"t":"summary"}` 记录——它也是上下文不是状态。v1a 只截断,UI 标「更早的对话未载入」。

## 8. 降级与可观测

- **诚实地板**:`complete` 抛 `LoomBackendError` → 流发 error 事件,面板给〔重试〕+「打开文件自己填」链接;留痕 `why=backend_error`。模型不在线时,书的一切照常手工可写——agent 是增强不是闸门。
- 留痕沿用 `.审稿留痕/领航员留痕.md`(文件名不改,§13 命名公约),why 枚举:`backend_error | tool_unparsed`——**只收失败类**。落盘器的 filename 撞车拒绝**不进留痕**:那是正确行为,现场已由对话 jsonl 的 result 事件记录并当场告知作者。旧枚举(`unparsed/few_options/false_exhausted`)随卡片机退役停产,留痕文件里的历史条目照常可读(read-only legacy)。`doctor` 检查改名「伙伴近况」,计数口径逐字不变。
- **demo 罐头**:DemoBackend 领航员分支升级为脚本化多轮(开场→提问→提设定候选→收尾)。**轮数推断**:从每轮重拼进 `user` 的对话尾部数 assistant 事件——DemoBackend 保持无状态桩(它每请求新建实例,实例计数器活不过一轮)。免 key 试玩端到端可点、可拍板、可落盘(demo 内容自报 `(demo)`)。

## 10. 服务端与并发

- `POST /api/partner/say {root, text}` → ndjson 流(`assistant_text`/`tool_call`/`tool_result`/`proposal`/`error`/`done`)。
- `POST /api/partner/confirm {root, id}`;`POST /api/partner/new {root}`;`GET /api/partner/history?root=`(尾部,供 UI 重开渲染)。
- **锁的三级裁量**(critical:织章一跑几分钟,`write_lock` 是非阻塞独占锁、拿不到即 409):
  1. **say 完全不碰书写锁**——轮内读书文件走无锁纯读(与 `journey_state`/`GET /api/file` 同款);追加 `.伙伴对话/*.jsonl` 无书锁争用面(五道工序永不读写该目录)。say 只拿一把**独立的 per-root 伙伴轮锁**(非阻塞,占用→409 `partner_busy`),防两次并发 say 交错写同一 `当前.jsonl`;它是流式端点,锁的持有形态照抄 `/api/write` 先例——endpoint 先 acquire(409 发生在响应头之前),锁随流持有,流终结的 finally 里 release。
  2. **confirm(拍板落盘)与 new(归档改名)才走真正的 `write_lock`**(非流式)——与 `journey_answer` 现状一致。织章期间 confirm 409 是**正确行为**,UX 见 §2(确认键置灰)。
  3. v1b 开工序:前端直接走 `/api/write` 原路,partner 不叠加持锁(§5.4)。
- **后端选择**:对话是主体验,直接 `get_backend`(主模型)。注:`cheap_backend` 是分模型省钱通道(为复审/写后摘要等「评估/管 what」调用而设,早于卡片机;现役消费者:质检/去AI味复审、除虫/连续性、learn 评估、诊断),**通道本体与这些消费者一概不动**——退役的只是 journey 侧两处调用点(§9)。

## 12. 测试策略

- **循环离线可测**:`ScriptedBackend`(回复序列)驱动多轮——断言工具执行顺序、jsonl 追加、≤6 次上限、解析失败回喂、终结条件。零真实模型。
- **流式纪律**:ScriptedBackend 以**刻意行中截断**的 chunk 序列(`用` 与 `:` 分两个 chunk)驱动,断言 `assistant_text` 事件不含任何协议行、工具块完整送达解析器;demo 罐头含工具块的轮次纳入同一断言。
- **前缀稳定(弱保证口径)**:书文件不变时,同一对话两次 `assemble` 的前缀 sha 相等;改 `agents/领航员.md` 或增删 `skills/` → 前缀变;改正文/外置大脑 → 只动动态后缀。
- **落盘定址**:三期蓝本 §10 的 land 测试表照单实现(filename 重命名解锁门禁/撞车拒绝/row 填冒号后/括注不剥),外加 proposal 生命周期三条:过期拒绝/重复确认幂等/快照失配拒绝。
- **落盘测试迁移**:`test_land_*` 全套的**断言语义逐条迁移**为落盘器新入口的测试(驱动保留的 `_replace_h2_body` 等通道,断言一字不放宽);替代先行,旧 `land_answer` 卡上下文测试随退役删除。**迁移期间任何一条语义断言消失 = 回滚,禁止「顺手简化」。**
- **上下文≠状态**:删 `.伙伴对话/` 后 `journey_state`/门禁/写章全部不变的回归测试。
- golden:伙伴稳定前缀进 golden 白名单(sha 人工审)。

## 15. 诚实的取舍

1. **起书变慢变贵,且丢掉两条结构保证**:多轮对话替代四段访谈(每段封顶 4 题,走 cheap 通道;旧流程 sections/card_lines 落盘各另烧一次消化调用,新旧成本差比表面小)。真正被静默换掉的是:(a) **预填 2-4 选项一键拍板**退化为打字作答或单候选确认(候选卡只承接单提案);(b) **`_MAX_QUESTIONS=4` 的硬收敛保证**消失,访谈长度取决于 persona 调教——两条都是「交互从简」的结构件,现在变成了 prompt 假设。缓解:环境快照让伙伴直奔空格;留痕收真机轮数数据;若实测收敛差,persona 可要求伙伴一题给 2-4 个候选(候选卡承接多提案留作 v2)。
2. **完全依赖模型在线**:离线起书退化为手填文件。地板是诚实的,但比卡片机时代矮。
3. **CLI 后端无缓存收益**:Claude/Codex CLI 每轮整段重发。可后续对 CLI 后端收紧对话尾预算,v1 不做。
4. **文本协议的工具遵从率是未验证假设**:CLI 护栏的结构性冲突已由伙伴变体解决(§3),剩余风险是模型对协议的遵从度。留痕体系收 `tool_unparsed` 数据;若真机很差,退路是对 OpenAI 兼容后端加原生 function calling 快速路径(注册表结构不变,只换传输),文本协议降为 CLI/demo 兜底。
5. **串行工具延迟**:每工具一次模型往返,轮内上限 6 次;伙伴「查三个文件再回答」可能要十几秒。流式(按 §5.2 纪律)让等待可见。
6. **一本书一条对话**的形态假设未经真机验证;归档/新开一段是逃生口,多会话列表留给后续。
7. **织章期间不能拍板**(锁裁量的结果):对话照聊,落盘要等——这是「文件写入互斥」的正确代价,UI 必须把等待说清楚而不是报错。
