# 起书完整性硬门禁 + 导入诊断回补 —— 设计

2026-07-12 · 已与作者逐节确认,经四视角对抗审核(红线兼容/架构/导入诊断落地/产品UX)加固。

## 0. 一句话

给 Loom 加一条**起书完整性硬门禁**:没把「立项+世界观+主角+前几章纲」填到线上,就**不给写第一章**(理由:没上下文交给 AI = 烂稿)。配套:已写好的小说可导入 → LLM 读正文提炼候选预填 → 残缺回领航员补齐 → 同一条线解锁。领航员补一个 IP 形象,在被拦场景说话。

## 1. 需求(已拍板)

1. **硬门禁**:起书完整性不达标就无法开写第一章。明确接受**反转 ADR 0011「缺卡/空卡照常出稿」**(立项"只读不回写"红线保留)。
2. **完整性标准线** = 立项 + 世界观 + 主角 + 前几章纲 四项齐(解锁写第一章的最小上下文全集)。
3. **导入诊断回补**:导入已写小说 → 整书诊断 = LLM 读正文把已有主角/世界观提炼成候选**预填**(作者确认)→ 真缺的回领航员访谈 → 同一条线解锁。诊断深度=读正文提炼预填,不是只打勾。
4. **领航员 IP 形象**(独立小子项目)。

## 2. 架构裁决:journey 当唯一完整性权威(方案1,经审核加固)

三个候选:①**journey.py 当唯一完整性权威 + write_precheck 单点拦 + importer 机械吞正文 + diagnose.py 独立跑 LLM(采纳)**;②独立 `gate.py`(否决:完整性双定义必漂,且 `gate` 一词在库内=质量关卡「绝不阻断」,硬阻断模块同名=词表污染);③新书也走诊断引擎(否决:空书无正文可读,纯烧 LLM,新书本有领航员引导)。

采纳①的加固论据(比初版更硬):`stage_done` 已是纯文件派生谓词,`writing_unlocked` 只是 AND 组合、零新状态,守 ADR 0013;`usecases` 已 `import journey`,`write_precheck → writing_unlocked` 是顺向新边、无环;`write_precheck` 是 CLI/server 双端唯一拦截点(usecases 模块注释明写它为防双入口漂移而设,learn 曾双写漂移有前科);**门禁绝不下沉 run_pipeline**(否则 golden/流水线测试全炸,且违单点)。

## 3. 完整性线(writing_unlocked,住 journey.py,纯文件派生)

| 项 | 谓词(吸收审核修正) |
|---|---|
| 立项 | 四格(分区/题材/对标意图/为什么选它)任一实质 **或整卡有实质非模板内容**(救导入的 `## 来自:` 头,`is_substantive` 剥模板);建书框选的题材由 scaffold 代落「题材」格(平台行先例)→ 新书建完即过 |
| 世界观 | 任一文件实质(现状 stage_done 不变) |
| 主角 | **存在「主角·名字」命名且非占位的卡**(复用专名册 `_NAME_SEP`+`_is_blank_or_template` 同源口径);单文件形态=有「## 主角 · 名字」带分隔符的实质 H2。**只填反派卡不算过** |
| 章纲 | **门禁阈值 = 卡章纲有 ≥1 实质章纲行(`- 第N章:` 命中)或整体有实质内容**(后者救段落式章纲,兑现导入「段落式照读不影响」承诺);**不要求 ≥3 行、不硬查下一待写章的行**(插空章会让行号错位→逐章硬查即误锁)。「K=3/从下一待写章起」只是**领航员出题目标**(问哪几章),按 next_chapter 参数化、不写死「前5章」——与门禁阈值分开 |

`writing_unlocked(root) -> (bool, missing: list[str])`,missing 用 STAGES key。

## 4. 门禁(住 write_precheck)

- **作用域(已拍板)= 无 Loom 织章即拦**:本书没有任何带 `.原稿` 快照/ledger 的章时才启用门禁。新书第一章拦、导入书首次让 Loom 续写拦、写到一半的老 Loom 书全豁免、样例书豁免(有 Loom 织的章)。判据仓内现成(snapshot_path/ledger 存在性),纯文件派生、零迁移、存量零伤害。
- **force 不越门禁**:门禁检查放 force 短路(`if force or not exists: return None`)**之前**;force 只豁免"已存在/漂移",不豁免完整性。
- **一次性解锁**:四项齐即开,不逐章硬查(插空章会让卡章纲行号错位——逐章硬查=从此每章被拦;逐章缺行提醒归二期章前决策卡)。
- **错误契约**:`409 {"error", "code": "brain_incomplete", "missing": [...], "stage": 首缺项}`。前端**按 code 分支**(brain_incomplete → 引导卡「去补齐」+ `goto` 首缺段 + 清面板 dismiss 标记;**绝不再渲染成"已存在"+覆盖按钮**);`cli.py` 拒绝理由字典补该 code(否则 KeyError 崩栈),CLI 文案给**文件路径出路**(CLI 无访谈通道)。
- **收编 brain_ready(否则"唯一权威"是空话)**:`project_state` 改暴露 `writing_unlocked + missing[]`(替代/并列 brain_ready);`app.js` 软拦(现 `showGuide`+「就这样写」ghost)升级为门禁引导卡,**删「就这样写」逃生门**(硬门禁下它必撞 409);brain_ready 若留旅程卡用途,注释标明是"铺过底"弱判据、非门禁判据。前端「写下一章」按钮按 `writing_unlocked` 提前置灰/先弹引导,正常路径永不走到 409(server 侧照拦兜底)。

## 5. skip 语义修正(门禁自洽)

- 四个门禁段**禁 skip**:`goto` 拒绝对它们 `skip=True`,前端「跳过这段」改「先放放」(只换段聚焦、不写 skip 标记);voice 独享真跳过。
- 领航员【无题】自动跳段(`next_card` exhausted 分支写 `skips[cur]`)**只在该段 stage_done 为真时生效**;为假时改出降级卡(防便宜模型对空文件误判无题 → 段被跳、谓词仍假、门禁仍锁、面板无题的死锁)。
- 面板常驻「距开写还差 N 项」;被拦引导卡 primary 位 = 「AI 铺底稿」(一次点击过世界观/人物/章纲三项)+ 题材建书代落立项 → 最短通过成本≈现状一次点击,"硬"只留给什么都不肯给的用户。

## 6. 导入吞正文(importer 扩展,零 LLM 红线不动)

- **入口**:欢迎页新入口「导入写好的小说」(与「导入 Loom 书」区分名),走既有 scan→commit。
- **正文桶**:BUCKETS 加「正文」,正则前置路由(stem 命中 `第[0-9一二三…]+章` 或纯序号→正文;撞其他桶关键词时关键词优先;猜不中让作者指认)。
- **`.txt` 只对正文桶放行**(读容错、落成 .md;设定桶维持 md-only)——已写小说多是 txt。
- **机械重命名**:正文桶强制归一 `第N章.md`(中文数字确定性换算;重号/缺号在确认屏按文件序重排 + 展示「原名→章号」映射)——否则 `第一章.md` 炸 `chapter_numbers`(`int(stem[1:-1])`)。内容原样(复用 `_write_dir_file` UTF-8 直拷/GBK 转码)。
- **v1 边界:一文件一章**;单大 txt 给降级话术。v1.5 做"按作者自己的显式章节标题确定性切分";**LLM 选切点永远归拆章工具**(backlog 三未决叉一个不背)。
- **导入小结补三句**:N 章已入库;导入章不能 learn 属正常(无 AI 原稿可 diff);建议对最近几章跑除虫铺状态账本(可选)。
- **配套无须新代码(审核实证)**:导入章无 ledger → `chapter_drifted` 恒 False;「改过」徽标恒 False;learn 对无快照章已有友好拒绝话术——不会一导入就处处报警。

## 7. 整书诊断(diagnose.py,scan→commit)

- **触发**:进书后伙伴面板独立动作「从正文提炼设定」——仅当正文有章**且**有未达标段时出现。**不是导入的一步**(importer 零 LLM 不动)。
- **scan(1 次 LLM)**:确定性采样**前 3 章全文 + 最近 2 章全文**(不读整本,约 4-5 万字,远在窗内);cheap_model 通道(提炼=评估管 what,不沾指纹/voice);产出三段候选(世界观小节/人物卡/章纲行),仿 draft 三段分隔、复用 `split_brain_draft`;**逐条带正文出处**(第N章短引,同 continuity BugItem evidence)——有出处=提炼,无出处=发明。**不落盘、无第二真相**,重启丢了重扫。调用前一句成本透明提示,跑长走 ndjson 流式进度。
- **commit(确认预填)**:候选走**一屏勾选清单**(复用导入路由确认形态,不逐题过);勾选/改写后直落——复用旅程 `_land_field/_land_sections/_land_card_lines`(**跳过 `_digest` 二次调用**;`_land_sections` 撞已有人写文件时诊断候选**丢弃**不追加——AI 提炼丢了可重扫,访谈答案才绝不丢)。
- **主角指认**:候选人物卡带一步「哪张是主角?」→ 确认改名 `主角·名字.md`(主角谓词在此兑现;否则导入书主角卡叫「林潜人设.md」永锁)。
- **立项段永不进诊断**(正文无平台/对标意图,提炼必编造);立项走访谈/手填/建书代落,ADR 0011「不回写」对诊断继续成立。
- **回补汇合(零额外机制)**:候选落盘 → `journey._sig` 失配 → 缓存卡作废 → 领航员按新现状只问真缺的 → 四项齐 → 门禁开。诊断=给访谈抢跑,两路汇合在同一条线。
- **降级**:scan 失败/解析不出→无候选、回纯访谈;部分解析→落能落的;guard(DRAFT_SECTION)逐段校验照旧。

## 8. 领航员 IP 形象(独立小子项目,随一期同发)

- **定位**:被拦是这个 IP 最高频出场(每本新书必遇一次)——门禁话术由**领航员人设**说("陪你把地基打完"),绝不用系统报错腔。
- **v1 交付**:一张领航员头像(与既有五工序头像同风格同尺寸,进 `webui/agents/`)+ 伙伴面板头部挂头像 + 言语风格统一进 `领航员.md`(温和、陪跑、不判罚)。视觉素材用图像生成出候选给作者挑。
- **刻意不做**:吉祥物动画/表情包/加载动效(违纸墨克制)。

## 9. 分期

- **一期(门禁,独立可发版)**:`writing_unlocked` 谓词 + `write_precheck` 接线(force 之前)+ 错误契约 + 前端引导卡收编 brain_ready + skip 修正 + scaffold 题材代落立项 + 样例书补立项卡 + 文案六处改口 + ADR 0014 + 领航员头像。结束即新书硬门禁完整生效。
- **二期(导入+诊断)**:正文桶 + 机械重命名 + `.txt` 放行 + `diagnose.py` scan/commit + 主角指认 + 一屏确认 + 回补汇合 + ndjson 进度。
- v1.5(可选):显式标题切分单大 txt。

## 10. 留档(新 ADR 0014)

「起书完整性硬门禁」:supersede ADR 0011「缺卡/空卡照常出稿」条款(立项"只读不回写"保留,建书代落/访谈代落除外);写死「**完整性门禁 ≠ 质量关卡,ADR 0006 一字不动**」(门禁是生成前置条件,非生成后质量判罚)防日后"Loom 从不阻断"民间传说反推门禁;记作用域谓词(无 Loom 织章)、force 语义、「门禁只住 write_precheck 绝不下沉 run_pipeline」纪律、诊断四条边界(来源限定带出处/确认前不落盘/复用人写优先通道/立项除外)、谓词无状态的已知行为(写到第20章删空世界观会重锁,与"改md签名失配重跑"哲学一致)。

## 11. 测试

- `writing_unlocked` 四项谓词 + 作用域(有无 Loom 织章)+ force 语义,pytest 钉死;conftest 加 `ready_project` 夹具(四项齐)。
- 正文桶重命名/中文数字换算/重号重排单测;`.txt` 放行仅正文桶。
- 诊断解析降级 fixture;主角指认改名。
- 既有 `test_usecases.py:61`(裸书 precheck is None)、`test_server_write_lock.py`(裸书 POST write)两处必炸用例同步改。
- golden/流水线测试**零改动验证**(门禁不下沉 run_pipeline);demo/CLI 冒烟剧本改用样例书或先铺四项。

## 12. 刻意不做(Avoid)

- 门禁下沉 run_pipeline(炸测试、违单点);逐章硬查章纲行(插空章即误锁);豁免位存 state(第二真相,违 0013)。
- AI 自动写立项卡(0011「不回写」保留;诊断也不碰立项)。
- 诊断直接 atomic_write 外置大脑(必须先出候选、作者确认才落;否则 importer「不重塑他的料」红线复活)。
- importer 里调 LLM(零 LLM 红线);LLM 选正文切点(归拆章工具,不背 backlog 三叉)。
- 给门禁段留 skip 后门(硬门禁留后门=没门);主角上 LLM 判别(易误伤,用文件名谓词)。
- 吉祥物动画/表情(违纸墨克制)。
