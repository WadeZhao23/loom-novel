# 调试 Loom

给「改代码 → 立刻看效果」用的开发者手册。**铁律:调试永远从源码跑,不碰装好的 `/Applications/Loom.app`**——`.venv` 是 editable 安装,源码即最新,那个打包 app 只给「当普通用户用」和「验打包 bug」。

## 0. 一次装好

```bash
cd ~/Desktop/Project/playground/Loom
python3 -m venv .venv                 # 已有就跳过
.venv/bin/pip install -e ".[dev]"     # editable + 测试依赖
.venv/bin/python -c "import loom; print(loom.__version__)"   # 确认能 import
```

入口脚本(装完在 `.venv/bin/`):`loom-app`(原生窗口)、`loom-serve`(浏览器兜底)、`loom`(引擎调试 CLI)。

## 1. 起服务(三选一)

| 场景 | 命令 | 说明 |
|---|---|---|
| **调 UI / 看形态**(最常用) | `LOOM_DEMO=1 .venv/bin/loom-serve` | 免 key,DemoBackend 出假内容但**跑真流水线**,浏览器自动开 |
| **调引擎/架构**(改 Python) | `LOOM_DEMO=1 .venv/bin/python -m uvicorn loom.server:app --port 8792 --reload` | `--reload`:存 `.py` 自动重启 |
| **验真实成品** | `.venv/bin/loom-app` | 带你的真 key、原生窗口,最接近发布态 |

- `LOOM_DEMO=1` → `backends.get_backend` 走 `DemoBackend`,不联网不花钱,但**五道工序/ledger/gates/events 全走真的**——测流程/架构够用。去掉它才用真模型。
- 只监听 `127.0.0.1`;端口自己定,别和别的实例撞。
- Claude Code 里:`.claude/launch.json` 已配 `loom-server`(8790 真)/`loom-demo`(8791 免key)/`loom-dev`(8792 免key+reload),用 preview 工具按名启动。

## 2. 改完怎么生效

| 改了什么 | 怎么生效 |
|---|---|
| webui(`loom/webui/*.js/css/html`) | **存盘 → 刷新浏览器**,不用重启(静态服务) |
| Python(引擎/server/journey…) | **重启服务**;用了 `--reload` 就自动重启 |
| agents 模板(`loom/templates/agents/*.md`) | 影响新建的书;老书读的是自己项目里 `agents/*.md`,得同步改或重 init |

## 3. 跑测试

```bash
.venv/bin/python -m pytest -q                    # 全量(现 392)
.venv/bin/python -m pytest tests/test_journey.py -v   # 单文件
.venv/bin/python -m pytest -k "land_answer"      # 关键字筛
.venv/bin/python -m evals.run_eval --gate        # 离线 eval 回归门禁(零 key)
```

golden 快照变更要人工审:`LOOM_GOLDEN_WRITE=1 .venv/bin/python -m pytest tests/test_golden_pipeline.py` 后 `git diff` 看白名单再提交。

## 4. 两条数据安全纪律(踩过坑)

1. **preview 浏览器和桌面端共享 localStorage**,`autoReopen()` 会自动打开上次那本书——曾误开真书《崇祯》并写进去。**调 UI 前先把 `loom_root` 钉到测试书**,或用 demo + 一本 scratchpad 空书;别在真书上点「出题 / 写章 / learn」。
   ```js
   // 浏览器 console:换到测试书
   localStorage.setItem('loom_root', '/绝对路径/测试书'); location.reload();
   ```
2. **一切写盘走原子写 + guard**,但调试期误触发的写入是真写。测破坏性路径(删章/learn/落盘)用**新建的 scratchpad 书**,不用真书。

## 5. 架构地图(找地方改)

一次 `loom write N` 的流水线主线:

```
server.py  POST /api/write ──→ usecases.write_chapter ──→ agents.run_pipeline
                                    │(per-root 写锁)          │
                                    │                          ├─ STEPS 五工序:设定师→大纲师→写手→编辑→润色师
                                    │                          │   每棒读「整个工作区」(累积,非纯链式)
                                    │                          ├─ guard.guard_write  落盘安全闸(非空/最小实字/结构标记)
                                    │                          ├─ ledger/resume      续跑账本(签名失配才重跑)
                                    │                          ├─ gates              质量关卡(质检挂编辑后/去AI味挂润色师后)
                                    │                          └─ events             ndjson 进度流(fire-and-forget)
                                    └─ 落 正文/第N章.md
```

| 想改什么 | 看哪 |
|---|---|
| 工序顺序 / 每棒读什么 | `loom/agents.py`(`STEPS` 表、`run_pipeline`、`_knowledge_items`) |
| 某个 agent 的系统提示词 | `loom/templates/agents/<角色>.md`(新书) / 项目 `agents/<角色>.md`(老书) |
| 模型路由 / 加供应商 | `loom/backends.py`(`PROVIDERS` 表、`get_backend`、`DemoBackend`) |
| 落盘校验 | `loom/guard.py` |
| 续跑逻辑 | `loom/resume.py` + `loom/ledger.py`(坏了当无、从头重跑) |
| 质量关卡 / 回炉 | `loom/gates.py` + 检测器 `aitell.py`/`fatigue.py` |
| 伙伴面板(答题起书) | `loom/journey.py`(状态机)+ 领航员 `templates/agents/领航员.md` + server `/api/journey/*` |
| HTTP 端点 | `loom/server.py`(薄壳)+ `loom/usecases.py`(编排 + 锁) |
| 跨会话状态 | `.loom_state.json`(`loom/state.py`:fingerprint_source / learned / journey 游标) |
| 前端 | `loom/webui/app.js`(全局 `DATA`、`jreq`、`render`)/ `style.css` / `index.html` |

红线(改前先读对应 ADR,`docs/adr/`):流水线安全攸关行为只住代码侧表不下放 frontmatter;md 是唯一真相;写作指纹只从你的改稿学;向前规划归人。

## 6. 追一次流程 / 定位 bug

- **看引擎在干嘛**:织章走 ndjson 事件流。preview 里 `read_network_requests` 看 `/api/write` 的流;或直接 CLI `LOOM_DEMO=1 .venv/bin/loom write <N> --root <测试书>` 看终端事件。
- **看前端报错**:preview `read_console_messages` / `read_network_requests`;`javascript_tool` 取运行时状态(如 `JOURNEY`、`DATA.root`)。
- **看服务端日志**:preview `preview_logs`;或前台起服务直接看 stdout。
- **CLI 单步调某一环**:`.venv/bin/loom --help` 看子命令(write / learn / seed / 拆书 / debug 等),不必走 UI 就能压某一棒。
- **复现测试失败**:先 `pytest -k <名> -x -q` 缩到单例,再读对应模块。系统性 bug 走 `superpowers:systematic-debugging`。

## 7. 什么时候才碰打包 .app

只有怀疑「打包后才现的 bug」——PyInstaller 无终端 `sys.stdout=None` 闪退、字体/资源没进包、hidden-import 缺失——才 build 一个跑:
```bash
.venv/bin/pip install -e ".[dist]"
.venv/bin/pyinstaller --noconfirm --clean packaging/loom.spec   # 产物在 dist/Loom.app
```
平时调试一律源码,不碰它。
